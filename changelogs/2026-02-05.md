## 2. Removed Star Rating System

### Overview
Removed star ratings from routes to avoid disincentivizing properties whose routes may not have high ratings.

### Changes
- Removed star rating display from route header
- Removed rating count display
- Routes now focus on engagement metrics (Likes, Completions, Shares) instead

### Rationale
Star ratings can negatively impact properties if their associated routes receive lower ratings. The engagement-based metrics provide value indication without the negative implications.

## 11. Address Search with Location-Based Route Finding

### Overview
Added the ability to search for routes by address/location. Users can now type an address and find routes within a specified radius.

### How It Works

**Step 1: Search for a Place**
1. Click the search bar
2. Select "Places" tab
3. Type an address, postcode, or place name
4. Mapbox Geocoding API returns matching places

**Step 2: Find Routes Nearby**
1. Click a place from the suggestions
2. System automatically searches for routes within 25km
3. Results sorted by distance from selected location

**Step 3: Create Route Option**
- If no routes found, users are offered a "Create a route here" button
- Takes them directly to route creation at that location

### New API Endpoints

**`/api/geocode/mapbox`**
- POST endpoint for geocoding addresses
- Uses Mapbox Geocoding API
- Biased to UK results
- Returns place name, coordinates, and type

**Updated `/api/routes/search`**
- New parameters: `lat`, `lng`, `searchRadius`
- Calculates distance using Haversine formula
- Returns `distanceFromSearch` for each route
- Results sorted by proximity

### UI Changes

**Mobile Search Overlay:**
- "Places" tab shows address suggestions as user types
- "Routes" tab shows routes near selected location
- Selected place shown in green banner at top
- Route cards show distance from search location
- "No routes nearby" state with create option

### Technical Details

**Geocoding Flow:**
```
User types "Worcester" вҶ’
API: POST /api/geocode/mapbox { query: "Worcester" } вҶ’
Returns: [{
  name: "Worcester",
  coordinates: { lat: 52.19, lng: -2.22 },
  type: "place"
}]
```

**Route Search Flow:**
```
User selects Worcester вҶ’
API: POST /api/routes/search { lat: 52.19, lng: -2.22, searchRadius: 25 } вҶ’
Returns routes within 25km sorted by distance
```
