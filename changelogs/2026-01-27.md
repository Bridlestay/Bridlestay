### 1. Fixed User Signup RLS Error

**Why:** New users couldn't sign up - got error: `new row violates row-level security policy for table 'user_stats'`

**Root Cause:**
- When a new user signs up, a database trigger tries to create a `user_stats` row
- The RLS policy requires `auth.uid() = user_id` for INSERT
- During trigger execution, this check fails because the auth context isn't properly set

**Files created:**
- `supabase/migrations/062_fix_user_stats_signup.sql`

**Changes:**
- Made `on_user_created_init_stats()` trigger function use `SECURITY DEFINER` to bypass RLS
- Relaxed INSERT policy on `user_stats` to allow initial row creation
- Added policy for service role operations

**Migration Required:**

Run this SQL in Supabase SQL Editor:

```sql
-- Recreate the function with SECURITY DEFINER
CREATE OR REPLACE FUNCTION on_user_created_init_stats()
RETURNS TRIGGER AS $$
BEGIN
  INSERT INTO user_stats (user_id) VALUES (NEW.id)
  ON CONFLICT (user_id) DO NOTHING;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

-- Allow inserting user_stats during signup
DROP POLICY IF EXISTS "Users can insert own stats" ON user_stats;
CREATE POLICY "Users can insert own stats" ON user_stats
  FOR INSERT WITH CHECK (true);

-- Ensure the trigger exists
DROP TRIGGER IF EXISTS trigger_user_created_init_stats ON users;
CREATE TRIGGER trigger_user_created_init_stats
  AFTER INSERT ON users
  FOR EACH ROW
  EXECUTE FUNCTION on_user_created_init_stats();
```

### 2. Fixed Email Confirmation Redirect (Supabase Config)

**Why:** After signing up, clicking the email confirmation link showed `{"error":"requested path is invalid"}` because Supabase was redirecting to `supabase.co/bridlestay.vercel.app` instead of `https://bridlestay.vercel.app/auth/callback`.

**Root Cause:** 
- Supabase Dashboard Site URL was missing `https://` protocol
- The Site URL needs to be the full URL including protocol

**Fix Required (Supabase Dashboard):**

Go to **Authentication → URL Configuration** and set:
- **Site URL**: `https://bridlestay.vercel.app` (must include https://)
- **Redirect URLs**: `https://bridlestay.vercel.app/**`

After fixing, users need to request a new confirmation email as old links were generated with wrong settings.
