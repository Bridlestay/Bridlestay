# Changelog — 2026-03-28

## Session 1

### Summary
Unified all 4 dropdown menus (main header, routes panel, routes map, mobile) into a single shared `UserDropdownMenu` component with an Airbnb-inspired clean design. Removed logo, user info header, and red logout text. Added clear section grouping, generous spacing, thin-stroke icons, and consistent styling across all pages.

### Changes

#### Shared UserDropdownMenu component
- **File created**: `components/user-dropdown-menu.tsx`
- **What changed**:
  - Single shared component renders `DropdownMenuContent` with all menu items
  - Airbnb-style design: `w-60`, `rounded-xl`, `shadow-xl`, clean section dividers
  - Items use `gap-3 px-4 py-2.5` for generous spacing
  - Icons: `h-[18px] w-[18px]` with `strokeWidth={1.5}` for thin, clean look
  - 5 clear sections separated by dividers: Notifications/Messages/Favorites, Routes/Listings/Admin, Profile/Settings/Help, Feedback/Claims, Log Out
  - No user info at top (removed name, email, role subtitle)
  - No logo at top (removed padoq branding from dropdown)
  - Log Out is plain text (removed red color)
  - Props: `user`, `unreadCount`, `notifCount`, `align`, `showRoutesLink`
  - Handles both logged-in and logged-out states
  - Contains its own logout logic (sign out + redirect)

#### Replaced 4 duplicate dropdown implementations
- **Files modified**: `components/header.tsx`, `components/routes/routes-panel-header.tsx`, `components/routes/routes-map-header.tsx`, `components/routes/mobile-top-header.tsx`
- **What changed**:
  - All 4 files now import and use `<UserDropdownMenu />` instead of inline dropdown content
  - Removed ~100 lines of duplicate menu items from each file
  - Removed unused `handleLogout` functions from all 4 files
  - Removed unused imports (`Image`, `useRouter`, icon imports) from all 4 files
  - Each file keeps its own `DropdownMenuTrigger` (different trigger styles per location)
  - Routes pages pass `showRoutesLink` prop; main header does not
