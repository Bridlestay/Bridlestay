# Changelog — 2026-03-30-RR

## Session Summary
Custom Mapbox thumbnail style for clean route previews, map waypoint click triggers cascade animation.

## Changes

### Clean Mapbox Thumbnails
- **File modified**: `lib/routes/route-thumbnail.ts`
- Using `outdoors-v12` style (terrain colours) with `&attribution=false&logo=false` to remove Mapbox/OpenStreetMap branding text
- Thumbnails rendered via CSS `object-cover` which naturally crops edges — combined with no-attribution params, gives a clean route-on-terrain look
- Park labels may occasionally appear but are barely legible at minicard size (120x80)

### Map Waypoint Click → Cascade Animation
- **File modified**: `components/routes/route-detail-drawer.tsx`
- Clicking a waypoint marker on the map now switches to the Waypoints info tab and triggers the cascade expand animation (same as clicking on the elevation chart)
- Previously, the handler tried to scroll directly to the DOM element — which failed for hidden waypoints (beyond the first 3) because the element didn't exist yet
- Now feeds the waypoint ID into `chartTargetWaypointId`, letting the timeline component handle the cascade expand + progressive scroll to target
- Also fixes a bug where `onWaypointFocused()` cleared `initialWaypointId` before the timeline could read it
