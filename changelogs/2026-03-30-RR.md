# Changelog — 2026-03-30-RR

## Session Summary
Custom Mapbox thumbnail style for clean route previews, map waypoint click triggers cascade animation.

## Changes

### Clean Mapbox Thumbnails
- **File modified**: `lib/routes/route-thumbnail.ts`
- Using `outdoors-v12` style (terrain colours) with `&attribution=false&logo=false` to remove Mapbox/OpenStreetMap branding text
- Thumbnails rendered via CSS `object-cover` which naturally crops edges — combined with no-attribution params, gives a clean route-on-terrain look
- Park labels may occasionally appear but are barely legible at minicard size (120x80)

### Map Waypoint Click → Cascade Animation
- **File modified**: `components/routes/route-detail-drawer.tsx`
- Clicking a waypoint marker on the map now switches to the Waypoints info tab and triggers the cascade expand animation (same as clicking on the elevation chart)
- Previously, the handler tried to scroll directly to the DOM element — which failed for hidden waypoints (beyond the first 3) because the element didn't exist yet
- Now feeds the waypoint ID into `chartTargetWaypointId`, letting the timeline component handle the cascade expand + progressive scroll to target
- Also fixes a bug where `onWaypointFocused()` cleared `initialWaypointId` before the timeline could read it

### Selected Route Pin Excluded from Clusters
- **File modified**: `components/routes/routes-map-mapbox.tsx`
- When a route is selected, its pin is now excluded from the clustering source data
- Previously, the selected route's pin could get absorbed into a nearby cluster, hiding it behind a cluster number
- The selected route already has its own S/E markers — the clustering pin was redundant and confusing
- Added `selectedRouteId` to the clustering useEffect dependency array so pins re-cluster when selection changes

### Tighter Clustering Threshold
- **File modified**: `components/routes/routes-map-mapbox.tsx`
- Reduced `clusterRadius` from 80px to 30px — pins now need to be nearly overlapping to cluster
- Raised `clusterMaxZoom` from 11 to 14 — clustering stays active at higher zoom levels but only when pins are very close
- Previously, pins clustered too eagerly when zooming out just slightly

### Improved Thumbnail Visibility for Large Routes
- **Files modified**: `lib/routes/route-thumbnail.ts`, `components/routes/route-quick-card.tsx`
- Quick card thumbnail image size increased from 120x80 to 200x150 — more pixels for the route line
- Default route weight increased from 4 to 6 — thicker line stays visible when zoomed out for large routes
- API padding reduced from 30px to 15px — less wasted space around the route in the thumbnail
- CSS `object-cover` on the container handles the larger image gracefully

### Fix: Route X Button Sometimes Not Closing
- **File modified**: `app/routes/page.tsx`
- Added a deselection guard (`lastDeselectedAtRef`) — ignores `handleRoutePreview` calls within 300ms of `handleClosePreview`
- Root cause: when clicking X on the quick card, the route is deselected and its pin gets re-added to the clustering source — if the pin appears where the click happened, the map's pin click handler can fire and immediately re-select the route
- The guard prevents this race condition regardless of the source

### Fix: Route Line Stuck After Property Pin Click
- **File modified**: `app/routes/page.tsx`
- When clicking a property pin that overlaps a route pin, both click handlers fired — the route got selected (line drawn, S/E markers) but the property card was shown on top
- Closing the property card didn't clear the route state, leaving the route line stuck on the map with no way to dismiss
- `handlePropertyPreview` now clears `selectedRouteId`, `selectedRouteData`, `drawnRouteId`, and `highlightedRouteId`
- Property card `onClose` also clears any lingering route state
