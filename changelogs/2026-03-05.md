## Session 1: Edit Waypoint View Visual Overhaul

- `components/routes/edit-waypoint-view.tsx`
- `components/routes/route-detail-drawer.tsx`

**What Changed:**

## Session 2: Edit View Improvements + Map Toggle Fix

**Summary:** Fixed edit view sizing to match route card photo hero, added photo upload to waypoint editing, hid edit on Start/Finish waypoints, and fixed View All waypoint map toggle.

---

### Changes Made

#### 1. Edit View Sizing + Photo Upload

**Modified Files:**
- `components/routes/edit-waypoint-view.tsx` (rewritten)
- `components/routes/route-detail-drawer.tsx` (pass routeId)

**What Changed:**
- Header now uses `h-48 md:h-56` (192px/224px) matching route card no-photo placeholder height
- Header displays waypoint's first photo as background (like route card photo hero) with overlay
- If no photos: shows MapPin icon placeholder (same style as route card no-photo state)
- Back button styled identically to route card close button (top-left, `bg-black/40`)
- "Add photo" button in bottom-right of hero area (matching "See all" photo button position)
- Photo count badge in bottom-left
- Photos section below form fields showing 3-column grid with delete buttons on hover
- Full photo upload flow: file picker, validation (10MB, image only), upload to API, instant state update
- Photo delete with confirmation
- `routeId` now passed as prop from drawer for API calls

#### 2. Hide Edit on Start/Finish Waypoints

**Modified Files:**
- `components/routes/waypoint-timeline.tsx`

**What Changed:**
- Added `!isStart && !isFinish` conditions to `onEdit` and `onSuggestEdit` callbacks
- Start and Finish are virtual waypoints (derived from route geometry), not DB records
- Prevents showing edit/suggest buttons that would fail on these waypoints

#### 3. View All Waypoint Map Toggle

**Modified Files:**
- `app/routes/page.tsx`

**What Changed:**
- `handleEnterViewMode("waypoints")` now sets `showWaymarkers: true` in layer settings
- Previously only hazards mode synced its toggle (`showHazards: true`)
- Added exit effect: waypoints view mode auto-exits when `showWaymarkers` is toggled off in settings
- Matches existing hazards pattern for symmetry

#### 4. Owner Photo Upload API Fix

**Modified Files:**
- `app/api/routes/[id]/waypoints/[waypointId]/photos/route.ts`

**What Changed:**
- Route owner can now upload photos to their own waypoints without completing the route
- Previously ALL uploads required route completion, blocking the owner from adding photos during edit
- Non-owners still require route completion (community photos)
