#### Find route click ŌåÆ fly to map + mini card
- **File modified**: `app/routes/page.tsx`
- **What changed**: Created `handleFindRouteClick` function. User can then choose to tap into full details or go back.
- **Why**: Opening the full detail drawer from Find was too aggressive. Users should see the route on the map first and decide if they want more details.

#### Improve Android geolocation compatibility
- **File modified**: `components/routes/route-navigator.tsx`
- **File modified**: `app/routes/page.tsx`
- **What changed**: Added specific error handling for GPS error codes: code 1 (permission denied) shows a clear "allow location access" message, code 2 (position unavailable) shows a "check GPS is enabled" message, code 3 (timeout) falls back to `enableHighAccuracy: false` with longer timeouts (30s) for Android compatibility. Increased base timeout from 10s to 15s and maximumAge from 1s to 2-3s.
- **Why**: Android Chrome can time out with `enableHighAccuracy: true` on some devices. The fallback to lower accuracy ensures GPS works on more devices.

## Session 2

### Summary
Navigation quality overhaul based on field-testing: GPS snap-to-route within 15m, route grey-out for traveled portions, bigger user arrow, remaining distance only (no time/ETA), thicker route line, proper navigation completion cleanup. Bug fixes: removed desktop search bar, fixed completion screen mobile overflow, fixed difficulty emojis (3 not 4), improved route card metadata spacing, fixed locate-me interference with route creation.

### Changes

#### GPS snap-to-route within 15m
- **File modified**: `components/routes/route-navigator.tsx`
- **What changed**: Added `snapToRoute` function that projects raw GPS position onto the nearest route segment. When within 15m of the route, the displayed position is snapped to the route line (like Google Maps road snapping). Raw GPS is still used for distance calculations and off-route detection (50m threshold). Returns segment index for route progress tracking.
- **Why**: Field testing showed GPS jitter causing the user dot to wander off the path even while walking straight. Snapping within 15m keeps the user visually on the route.

#### Route grey-out (traveled portion)
- **File modified**: `components/routes/routes-map-mapbox.tsx`
- **File modified**: `app/routes/page.tsx`
- **What changed**: Added `navSegmentIndex` prop to the map component. During navigation, a separate "routes-traveled" GeoJSON source/layer draws the portion of the route behind the user in grey (#9CA3AF, 10px wide, 70% opacity). The traveled portion updates as the user progresses along the route. Cleaned up on navigation exit.
- **Why**: Field testing showed the route looked the same ahead and behind ŌĆö no sense of progress. Now traveled sections are visually distinct.

#### Bigger user arrow marker
- **File modified**: `components/routes/routes-map-mapbox.tsx`
- **What changed**: Increased user arrow from 32x32 SVG in 44x44 container to 48x48 SVG in 60x60 container. Thicker stroke (4px vs 3px), bigger pulse ring (3px border). Arrow path scaled proportionally.
- **Why**: Field testing showed the user arrow was too small to easily track on the map during navigation.

#### Bottom bar: remaining distance only
- **File modified**: `components/routes/route-navigator.tsx`
- **What changed**: Removed time remaining estimate, ETA clock time, and associated calculation functions (`formatTimeRemaining`, `formatETA`, `avgSpeedKmh`, `estimatedSecondsRemaining`). Bottom bar now shows only remaining distance (e.g., "2.3 km" or "450m") with "remaining" label below.
- **Why**: Field testing showed "< 1 min" was confusing and unreliable. Distance is the only metric that matters during navigation.

#### Thicker route line during navigation
- **File modified**: `components/routes/routes-map-mapbox.tsx`
- **What changed**: Increased navigation route line from 8px to 10px width, outline from 12px to 14px.
- **Why**: Field testing showed the route line wasn't prominent enough during navigation.

#### Proper navigation completion cleanup
- **File modified**: `app/routes/page.tsx`
- **What changed**: Navigation completion now resets `navSegmentIndex` to 0 in both the `handleNavigationComplete` callback and the manual close handler. The close handler also clears `navigatingRoute`.
- **Why**: Field testing showed the route staying open after completion instead of restoring to normal map view.

#### Remove desktop search bar
- **File modified**: `components/routes/routes-map-header.tsx`
- **What changed**: Removed search form, search state, `onSearch` prop, `handleSearchSubmit` handler, and unused imports (`Input`, `Search`, `Badge`, `Map`). Component signature simplified to `export function RoutesMapHeader()`.
- **Why**: Search bar was redundant with Find Routes panel search. Desktop now matches mobile ŌĆö hamburger menu + profile avatar only.

#### Fix completion screen mobile overflow
- **File modified**: `components/routes/post-ride-review.tsx`
- **What changed**: Added `max-h-[90vh] overflow-y-auto` to `DialogContent` so the completion screen scrolls on small screens instead of overflowing.
- **Why**: Field testing showed the completion dialog getting cut off on mobile screens.
