# Changelog — 2026-03-19

## Session 1

### Summary
Major navigation and UI restructure: simplified mobile/desktop nav to 3 tabs (Find, Record, Create), removed top search bar for floating hamburger + PFP, merged Saved routes into Find as 3rd tab, changed Find route clicks to fly-to-map with mini card, improved Android geolocation compatibility.

### Changes

#### Simplify mobile bottom nav — Find, Record, Create
- **File modified**: `components/routes/mobile-bottom-nav.tsx`
- **What changed**: Removed Map and Saved tabs from mobile bottom navigation. Now shows only: Find (left), Record (center, with special red indicator), Create (right). The map is always the default background — no tab needed to switch to it.
- **Why**: Map is always visible behind panels. Saved is now inside Find. 5 tabs was too many for mobile.

#### Simplify desktop nav — Find, Record, Create
- **File modified**: `components/routes/routes-nav-tabs.tsx`
- **What changed**: Removed Map and Saved tabs from the desktop nav pill in the top-right corner. Desktop now shows only Find Routes and Create Route tabs.
- **Why**: Same reasoning as mobile — map is the default view, saved is inside Find.

#### Remove top search bar — floating hamburger + PFP
- **File modified**: `components/routes/mobile-top-header.tsx`
- **What changed**: Removed the white header bar with search input. The hamburger menu and profile picture are now floating circular buttons with shadows (like Google Maps) — hamburger top-left, PFP top-right. The entire search overlay (Places/Routes tabs, geocoding, radius control) was removed since users search via the Find section. Header is now `pointer-events-none` with each button having `pointer-events-auto`.
- **Why**: The search bar was redundant with the Find section's search. Floating buttons give more map space and match modern map app patterns.

#### Remove Share button from Find Routes
- **File modified**: `components/routes/find-routes-panel.tsx`
- **What changed**: Removed the Share button and `handleShare` function from the Find Routes panel header.
- **Why**: Unnecessary feature — users don't need to share filter URLs.

#### Merge Saved Routes into Find as 3rd tab
- **File modified**: `components/routes/find-routes-panel.tsx`
- **File modified**: `app/routes/page.tsx`
- **What changed**: Added a "Saved" tab to the Find Routes panel alongside Recommended and All. When active, it shows My Routes / Bookmarked sub-tabs with chip toggles. Fetches from `/api/routes/my-routes` and `/api/routes/favorites`. Sort/filter controls are hidden on the Saved tab. Removed the separate `SavedRoutesPanel` import and usage from page.tsx.
- **Why**: Having a separate Saved tab in the nav was unnecessary — it's a subset of route browsing and belongs inside Find.

#### Find route click → fly to map + mini card
- **File modified**: `app/routes/page.tsx`
- **What changed**: Created `handleFindRouteClick` function. Instead of opening the full detail drawer, clicking a route in Find: closes the Find panel, switches to map tab, fetches full route data, flies to the route, and shows the quick card at the bottom. User can then choose to tap into full details or go back.
- **Why**: Opening the full detail drawer from Find was too aggressive. Users should see the route on the map first and decide if they want more details.

#### Improve Android geolocation compatibility
- **File modified**: `components/routes/route-navigator.tsx`
- **File modified**: `app/routes/page.tsx`
- **What changed**: Added specific error handling for GPS error codes: code 1 (permission denied) shows a clear "allow location access" message, code 2 (position unavailable) shows a "check GPS is enabled" message, code 3 (timeout) falls back to `enableHighAccuracy: false` with longer timeouts (30s) for Android compatibility. Increased base timeout from 10s to 15s and maximumAge from 1s to 2-3s.
- **Why**: Android Chrome can time out with `enableHighAccuracy: true` on some devices. The fallback to lower accuracy ensures GPS works on more devices.

### Files Modified
- `components/routes/mobile-bottom-nav.tsx` — 3 tabs: Find, Record, Create
- `components/routes/routes-nav-tabs.tsx` — 2 desktop tabs: Find Routes, Create Route
- `components/routes/mobile-top-header.tsx` — Floating hamburger + PFP, no search bar
- `components/routes/find-routes-panel.tsx` — Saved tab, no Share button
- `components/routes/route-navigator.tsx` — Android GPS fallback
- `app/routes/page.tsx` — Find click → map flyover, removed SavedRoutesPanel, Android GPS fallback

## Session 2

### Summary
Navigation quality overhaul based on field-testing: GPS snap-to-route within 15m, route grey-out for traveled portions, bigger user arrow, remaining distance only (no time/ETA), thicker route line, proper navigation completion cleanup. Bug fixes: removed desktop search bar, fixed completion screen mobile overflow, fixed difficulty emojis (3 not 4), improved route card metadata spacing, fixed locate-me interference with route creation.

### Changes

#### GPS snap-to-route within 15m
- **File modified**: `components/routes/route-navigator.tsx`
- **What changed**: Added `snapToRoute` function that projects raw GPS position onto the nearest route segment. When within 15m of the route, the displayed position is snapped to the route line (like Google Maps road snapping). Raw GPS is still used for distance calculations and off-route detection (50m threshold). Returns segment index for route progress tracking.
- **Why**: Field testing showed GPS jitter causing the user dot to wander off the path even while walking straight. Snapping within 15m keeps the user visually on the route.

#### Route grey-out (traveled portion)
- **File modified**: `components/routes/routes-map-mapbox.tsx`
- **File modified**: `app/routes/page.tsx`
- **What changed**: Added `navSegmentIndex` prop to the map component. During navigation, a separate "routes-traveled" GeoJSON source/layer draws the portion of the route behind the user in grey (#9CA3AF, 10px wide, 70% opacity). The traveled portion updates as the user progresses along the route. Cleaned up on navigation exit.
- **Why**: Field testing showed the route looked the same ahead and behind — no sense of progress. Now traveled sections are visually distinct.

#### Bigger user arrow marker
- **File modified**: `components/routes/routes-map-mapbox.tsx`
- **What changed**: Increased user arrow from 32x32 SVG in 44x44 container to 48x48 SVG in 60x60 container. Thicker stroke (4px vs 3px), bigger pulse ring (3px border). Arrow path scaled proportionally.
- **Why**: Field testing showed the user arrow was too small to easily track on the map during navigation.

#### Bottom bar: remaining distance only
- **File modified**: `components/routes/route-navigator.tsx`
- **What changed**: Removed time remaining estimate, ETA clock time, and associated calculation functions (`formatTimeRemaining`, `formatETA`, `avgSpeedKmh`, `estimatedSecondsRemaining`). Bottom bar now shows only remaining distance (e.g., "2.3 km" or "450m") with "remaining" label below.
- **Why**: Field testing showed "< 1 min" was confusing and unreliable. Distance is the only metric that matters during navigation.

#### Thicker route line during navigation
- **File modified**: `components/routes/routes-map-mapbox.tsx`
- **What changed**: Increased navigation route line from 8px to 10px width, outline from 12px to 14px.
- **Why**: Field testing showed the route line wasn't prominent enough during navigation.

#### Proper navigation completion cleanup
- **File modified**: `app/routes/page.tsx`
- **What changed**: Navigation completion now resets `navSegmentIndex` to 0 in both the `handleNavigationComplete` callback and the manual close handler. The close handler also clears `navigatingRoute`.
- **Why**: Field testing showed the route staying open after completion instead of restoring to normal map view.

#### Remove desktop search bar
- **File modified**: `components/routes/routes-map-header.tsx`
- **What changed**: Removed search form, search state, `onSearch` prop, `handleSearchSubmit` handler, and unused imports (`Input`, `Search`, `Badge`, `Map`). Component signature simplified to `export function RoutesMapHeader()`.
- **Why**: Search bar was redundant with Find Routes panel search. Desktop now matches mobile — hamburger menu + profile avatar only.

#### Fix completion screen mobile overflow
- **File modified**: `components/routes/post-ride-review.tsx`
- **What changed**: Added `max-h-[90vh] overflow-y-auto` to `DialogContent` so the completion screen scrolls on small screens instead of overflowing.
- **Why**: Field testing showed the completion dialog getting cut off on mobile screens.

#### Fix difficulty emojis (3 not 4)
- **File modified**: `components/routes/post-ride-review.tsx`
- **What changed**: Removed "Severe" difficulty option (😰). Now shows only Easy (😊), Moderate (🙂), Difficult (😤) — matching the route schema. Grid changed from 4 columns to 3.
- **Why**: Route creation only supports 3 difficulty levels. The review picker had 4, creating a mismatch.

#### Fix route card metadata spacing
- **File modified**: `components/routes/route-card.tsx`
- **What changed**: Reduced badge gap from `gap-2` to `gap-1.5`. Added explicit distance badge (e.g., "3.2 km"). Changed time badges from "🐴 Xm" to "🐴 X min" (was ambiguous — looked like metres). Added "gain" suffix to elevation. Removed Clock icon from time badges (emoji is sufficient).
- **Why**: Field testing showed metadata badges looking cramped and time values being ambiguous.

#### Fix locate-me interference with route creation
- **File modified**: `app/routes/page.tsx`
- **What changed**: The `startCreating` function now stops the locate-me GPS watch (`clearWatch`, reset `locateWatchRef`, `setIsLocating(false)`) before entering creation mode.
- **Why**: Field testing showed that if locate-me was active before starting route creation, the map kept re-centering on the user's location, making it difficult to plot waypoints.

- `components/routes/routes-map-mapbox.tsx` — Grey-out, bigger arrow, thicker line, navSegmentIndex prop
- `app/routes/page.tsx` — navSegmentIndex state, completion cleanup, locate-me fix
- `components/routes/routes-map-header.tsx` — Removed search bar

## Session 3

### Summary
Airbnb-style fair route recommendation system: removed Recommended tab, added Featured Routes section with scoring algorithm, built admin route management page with boost/suppress controls, impression tracking for fair rotation.

### Changes

#### Fair rotation scoring algorithm
- **File created**: `app/api/routes/featured/route.ts`
- **What changed**: New API endpoint that scores all public routes using a weighted algorithm:
  - Quality (30%): avg_rating + log-scale review_count
  - Freshness (20%): 14-day boost for new routes, decaying to 0 at 60 days
  - Exposure fairness (30%): inverse log of impression_count — routes shown less get boosted
  - Random jitter (20%): controlled randomness so different routes surface each load
  - Admin multiplier: 0x (hidden), 0.5x (suppressed), 1x (normal), 1.5x (boosted), 2x (featured)
- **Why**: Ensures all route creators get fair visibility, not just those with the highest ratings. New routes get a chance, and over-exposed routes naturally cycle out.

#### Database migration for scoring columns
- **File created**: `supabase/migrations/080_route_scoring.sql`
- **What changed**: Added `admin_boost_multiplier` (default 1.0), `impression_count` (default 0), and `last_featured_at` columns to routes table. Added composite index for scoring queries.
- **Why**: Required for the scoring algorithm and admin controls.

#### Remove Recommended tab, add Featured Routes section
- **File modified**: `components/routes/find-routes-panel.tsx`
- **What changed**: Removed "Recommended" tab — now just "All Routes" and "Saved" tabs. Added a "Featured Routes" horizontal scroll section at the top of the All tab showing 3 algorithmically-selected routes. Featured section hidden during search. Fetches from `/api/routes/featured?limit=3`.
- **Why**: "Recommended" was a stub (API ignored the param). Featured section provides real, fair recommendations visible at the top of All.

#### Admin route management API
- **File created**: `app/api/admin/routes/route.ts`
- **What changed**: GET endpoint lists all routes with scoring data (impression_count, admin_boost_multiplier, owner name), supports search/sort/pagination. PATCH endpoint allows admins to update boost multiplier (0-10) and reset impression counts. Admin-only auth checks.
- **Why**: Admins need to control route visibility and monitor fair distribution.

#### Admin route management UI
- **File created**: `app/admin/routes/page.tsx`
- **File created**: `components/admin/admin-route-manager.tsx`
- **What changed**: Full admin page with:
  - Summary cards (total routes, boosted, suppressed, hidden counts)
  - Searchable, sortable table of all routes with owner, rating, impressions, creation date
  - Dropdown to set boost level: Hidden (0x), Suppressed (0.5x), Normal (1x), Boosted (1.5x), Featured (2x)
  - Button to reset impression count (gives route a fresh chance)
  - Algorithm explanation panel at bottom
- **Why**: Admins need visibility into which routes are being shown and the ability to boost/suppress specific routes.

#### Admin menu links
- **File modified**: `components/routes/mobile-top-header.tsx`
- **File modified**: `components/routes/routes-map-header.tsx`
- **What changed**: Added "Route Management" link in admin dropdown menus for both mobile and desktop.
- **Why**: Admins need quick access to the route management page.

### Risk Notes

- The migration must be run on Supabase before the new columns work. Until then, the API defaults (1.0, 0, null) will be used.
