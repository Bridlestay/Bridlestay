# Changelog — 2026-02-22

## Session: Community-Cleared Warnings + Waypoint Tags

Two features based on partner feedback: warnings now require community votes to clear (instead of a single user instantly resolving), and waypoints gain purpose tags with filtering.

---

### Feature 1: Community-Cleared Warnings

Warnings can no longer be instantly resolved by a single user clicking "Cleared?". Instead, multiple riders must vote to confirm a warning is no longer an issue. The threshold scales with route activity (minimum 2 votes, up to 10% of recent completions in the last 90 days). Warnings expire after a fixed 30 days. After completing a ride, users are prompted about active warnings before seeing the thank-you screen.

### Feature 2: Waypoint Tags

Waypoints now have a `tag` field (instruction, POI, note, caution) separate from the existing `icon_type`. Tags show as color-coded badges on waypoint cards and in the waypoints panel. The panel has filter pills to show/hide waypoints by tag. Map markers are colored by tag.

### Risk Notes

- Vote count maintained by DB trigger to prevent race conditions
- `clear_votes_needed` is set at warning creation time and stored — won't change retroactively
- 30-day max expiry still works via existing `cleanup_expired_hazards()` function
- Waypoint `tag` defaults to `'note'` — all existing waypoints get sensible default, no data migration needed
- Warning check in review flow uses `warningCheckDone` gate on step 5 — no step renumbering needed
