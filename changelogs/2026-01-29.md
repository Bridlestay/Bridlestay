## 13. Trust Score System Documentation

**Status:** Trust score system is fully implemented and working.

**How it works:**
- Each user starts with a base score of 30
- Score increases with:
  - Account age (+1 per month, max +15)
  - Social login (+10)
  - Bookings made/hosted (+3 each, max +15)
  - Reviews given (+2 each, max +10)
  - Accurate reports (+5 each, max +10)
- Score decreases with:
  - False reports (-10 each, max -20)
  - Warnings received (-10 each, max -30)

**Trust Levels:**
- 90+: Moderator
- 75+: Verified (internal status, not displayed to users)
- 50+: Trusted
- 25+: Standard
- <25: New

**Where it's used:**
- Admin inspection dashboard shows trust score
- Moderation system uses trust to prioritize reviews
- Report credibility is weighted by reporter's trust score

**Files:**
- `supabase/migrations/050_moderation_system.sql` - Database schema and functions
- `app/api/moderation/trust/route.ts` - API endpoint
- `components/admin/inspect-dashboard.tsx` - Admin display

## 17. Admin Panel: Warning / Soft Ban / Ban System - DOCUMENTATION

### Current Implementation Status: ✅ FULLY IMPLEMENTED

The enforcement system IS fully implemented. Here's what each action does:

### **Warning (Implemented)**
- **What it does:**
  - Sends an educational message to the user's inbox
  - Records the warning in `user_warnings` table
  - Increments `warnings_received` count on user profile
  - Updates user's trust score (−10 per warning, max −30)
  - User can still access ALL features (messages, bookings, etc.)
- **User experience:** Receives a system message in their inbox with the warning
- **Visibility:** Warning shows in admin "Trust & Safety" panel
- **Files:** `components/admin/admin-moderation-queue.tsx` (sends warning)

### **Soft Ban (Implemented)**
- **What it does:**
  - Sets `softbanned = true` on user profile
  - Records reason and timestamp
  - Sends system message to user explaining the soft ban
  - **User CAN still:**
    - Log in
    - Read their messages
    - View their bookings
  - **User CANNOT:**
    - Send new messages
    - Create new bookings
    - Post reviews/comments
- **User experience:** Can still access inbox to read the ban message
- **Files:** `app/api/admin/actions/softban-user/route.ts`

### **Full Ban (Implemented)**
- **What it does:**
  - Sets `banned = true` on user profile
  - Records reason, timestamp, and admin who banned
  - Sends system message before ban takes effect
  - **User CANNOT:**
    - Log in at all
    - Access any features
- **Note:** User should receive email notification (if configured in Supabase)
- **Files:** `app/api/admin/actions/ban-user/route.ts`

### Database Tables Used:
- `users.banned` - boolean for full ban
- `users.softbanned` - boolean for soft ban
- `users.ban_reason` - text explanation
- `users.banned_at` - timestamp
- `users.banned_by` - admin who issued
- `user_warnings` - warning history
- `enforcement_actions` - full audit trail

## 18. Referral System - DOCUMENTATION

### How Referrals Work

**YES - Referrals DO track and DO stack (for the referrer):**

### For the Friend (person using the code):
- Gets a **fee discount** (configurable %, e.g., 10% off guest fees)
- Discount applies for a **limited duration** (e.g., 3 months)
- Discount applies up to a **limited number of bookings** (e.g., 5 bookings)
- **Does NOT stack** - a user can only redeem ONE referral code

### For the Referrer (person who shared the code):
- Gets rewards for **EACH** friend who uses their code
- **DOES stack** - if 5 friends use your code, you get 5 rewards
- Reward is typically a fee discount on their next booking
- Rewards tracked in `user_referral_rewards` table

### Tracking System:
- `referral_codes` - stores all codes
- `referral_redemptions` - tracks who used which code
  - `user_id` - who redeemed it
  - `referrer_user_id` - who referred them
  - `status` - active/expired/exhausted
  - `total_savings_pennies` - how much they've saved
  - `bookings_with_benefit` - how many bookings used the discount
- `referrer_rewards` - tracks referrer rewards

### Configurable Settings (Admin Panel → Referrals → Settings):
1. **Friend Benefit Type:** Guest fee discount % or Host fee waiver %
2. **Discount Percentage:** e.g., 10%
3. **Duration:** How many months the benefit lasts
4. **Max Bookings:** How many bookings can use the discount
5. **Referrer Benefit:** What the person who shared gets (% off their fees)

### Example Flow:
1. Alice generates referral code: `ALICE1234`
2. Bob signs up with code `ALICE1234`
3. Bob gets: 10% off guest fees for 3 months (up to 5 bookings)
4. When Bob makes his first booking: Alice gets 50% off guest fees on her next booking
5. Charlie also uses `ALICE1234` → Alice gets ANOTHER reward (stacks!)

### Current Default Settings:
- Friend: 10% off guest fees, 3 months, 5 bookings max
- Referrer: 50% off guest fees on next booking

### NO Cash Rewards:
- Fixed credit option was **removed** (as per previous request)
- Only fee discount percentages are supported
- This avoids needing a cash payout system
