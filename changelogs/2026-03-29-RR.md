# Changelog — 2026-03-29-RR

## Session Summary
Redesigned the waypoint elevation chart markers and added mobile swipe support for info tabs.

## Changes

### Elevation Chart Marker Redesign
- **File modified**: `components/routes/elevation-profile.tsx`
- All floating waypoint markers (S, 1, 2, ..., F) now use a unified dark green colour (`#267347`) matching the graph line — previously S was green, F was red, and numbered waypoints were blue
- Drop lines changed from coloured to soft grey (`#94A3B8`, 50% opacity) — subtle and non-distracting
- Drop lines now stop at the graph line intersection instead of extending to the bottom of the chart
- Intersection dots enlarged slightly (r=1.2) and coloured green to match the graph line

### Mobile Tab Swiping
- **File modified**: `components/routes/route-detail-drawer.tsx`
- Added touch swipe gesture support on the tab content area (Elevation, Waypoints, Hazards, Warnings, Variants, Weather)
- Swipe left/right to navigate between tabs on mobile
- Requires >50px horizontal movement and dominant horizontal direction to trigger
- Works alongside existing click-to-select behaviour

### Elevation Chart Drop Line Fixes
- **File modified**: `components/routes/elevation-profile.tsx`
- Drop lines now extend upward into the marker gutter (`y1=-20`) so they visually connect to the circles — previously started at `y=0` (top of chart SVG) leaving a gap
- Drop lines extended +1 SVG unit past `yOnLine` to visually touch the graph curve (Catmull-Rom spline overshoot caused a small gap)
- SVG set to `overflow: visible` so negative-y drop lines render into the gutter
- Marker gutter reduced from `h-14` to `h-10` — pushes circles closer to chart, cleaner look

### Elevation Chart Drop Line Refinements (v2)
- **File modified**: `components/routes/elevation-profile.tsx`
- Removed intersection dots at the bottom of drop lines — cleaner look
- Markers shifted down (`bottom: -12px`) so their center aligns with the top of the drop lines
- Drop lines extended further (`yOnLine + 2`) to visually meet the graph curve
- Gutter reduced to `h-8` for tighter spacing

### New Map Pin Images
- **File modified**: `components/routes/routes-map-mapbox.tsx`, `public/Pins/`
- New pin images (170x240) copied over old filenames (`route-pin.png`, `property-pin.png`) to avoid 404 issues with new filenames not being served by Next.js dev server
- Reverted code to reference original filenames — images are the new v2 designs
- Rendered at icon-size 0.6 with pixelRatio 2

### Chart Waypoint Click → Expand & Scroll
- **Files modified**: `components/routes/route-detail-drawer.tsx`, `components/routes/waypoint-timeline.tsx`
- Clicking a waypoint on the elevation chart that's hidden in the collapsed timeline now triggers the cascade expand animation and scrolls to that specific waypoint
- Added `chartTargetWaypointId` state in the drawer that passes through to `initialExpandedWaypointId`
- `handleExpand` now accepts an optional `scrollToId` — scrolls to the target after cascade completes instead of scrolling to the end
- Visible waypoints still scroll immediately without expanding

### Chart Click → Progressive Cascade Scroll
- **File modified**: `components/routes/waypoint-timeline.tsx`
- When clicking a hidden waypoint on the elevation chart, the scroll now follows the cascade animation down (same as normal expand) instead of waiting until the cascade finishes
- Scroll stops once the target waypoint is fully revealed — the cascade continues but the viewport stays parked at the target
- `startProgressiveScroll` now accepts an optional `targetId` — scrolls that element into center view each frame instead of following `endRef`
- Stop timing calculated from target's stagger index: `(targetIdx * STAGGER_DELAY) + CARD_DURATION + 200ms`

### Fix: "Show fewer" Collapse Glitch
- **File modified**: `components/routes/waypoint-timeline.tsx`
- `handleExpand` had `hiddenWaypoints` (a new array each render) in its dependency array, causing it to be recreated every render
- This triggered the auto-expand `useEffect` (which depends on `handleExpand`) to fire on every render — fighting the collapse animation
- Fixed by storing `hiddenWaypoints` in a ref (`hiddenWaypointsRef`) and reading from it inside the callback, removing the unstable dependency

### Pin Size Reduction
- **File modified**: `components/routes/routes-map-mapbox.tsx`
- Reduced route and property pin `icon-size` from 0.6 to 0.5 (~15% smaller)
