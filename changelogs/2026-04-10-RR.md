# 2026-04-10 — Homepage redesign + header refactor

## Session summary

Redesigned the homepage hero from a flat green gradient band into a contained-card photo hero, moved the main nav from the centre of the header to the top-right next to the avatar, fixed the "Become a Host" loading flicker for hosts and admins, swapped the logo to the new test logo, and reordered the homepage so Featured Properties sits directly beneath the hero.

## Files modified

- `app/page.tsx` — replaced green-gradient hero with photo hero (`/photo-hero.png`) inside a contained card with rounded corners, dark bottom gradient overlay, hero text + search bar overlaid. Reordered sections so Featured Properties now comes directly under the hero, with Why Choose padoq moved beneath it. Removed unused `Star` import.
- `components/header.tsx` — moved Search / Routes / Become a Host nav links from the centre into a flex group on the right next to the avatar / dashboard / hamburger. Logo source updated from `/logo.png` to `/logo-test.png`. Added `showBecomeHost` derived state that gates the "Become a Host" link on `!loading` so it never flashes for hosts and admins during the auth fetch.

## Files renamed

- `public/Test Logo.png` → `public/logo-test.png`
- `public/Photo Hero.png` → `public/photo-hero.png`

Filenames with spaces were causing awkward URL encoding (`%20`) and would cause friction in future code references. Renamed to kebab-case before any code referenced them.

## What changed and why

1. **Photo hero replaces green band** — the existing hero was a flat green gradient. The new design uses a real photograph of horses in a UK paddock, contained in a card with rounded corners (not full-bleed), matching the user-supplied reference image. A `bg-gradient-to-t from-black/70 via-black/20 to-transparent` overlay sits beneath the text to keep the white serif title and subhead legible against the bright photo.

2. **Header nav moved right** — Search / Routes / Become a Host were previously centred in the header (`flex-1 justify-center`). Wrapped in a flex group with the avatar / dashboard / hamburger so they all sit together in the top-right. The Padoq logo stays in the top-left.

3. **"Become a Host" flicker fix** — the previous gating expression `(!user || (user.role !== "host" && user.role !== "admin"))` evaluated to true while `user` was still `null` during the initial fetch, so hosts and admins briefly saw the link before it disappeared. Replaced with a `showBecomeHost` constant that requires `!loading`, so the link only ever renders once we know the user's role for sure. Logged-out visitors and guests still see it; hosts and admins never see it, even for a frame.

4. **Logo swap** — `/logo.png` → `/logo-test.png` per the new branding asset Richard dropped in `public/`.

5. **Section reorder** — Photo hero → Featured Properties → Why Choose padoq → Routes Teaser → Host CTA. Featured Properties moved up to sit immediately below the hero so the first thing visitors see after the search bar is real properties, not marketing tiles.

## Architectural decisions

- **Did not split Header into a server component + client island.** The original plan was to convert Header to an async server component that fetches the user via the server Supabase client, eliminating the flicker entirely. During implementation, grep revealed that 7 client-component pages import Header directly (`app/search/page.tsx`, `app/host/page.tsx`, `app/host/listings/page.tsx`, `app/routes/edit/[id]/page.tsx`, `app/account/edit/page.tsx`, `app/verify/complete/page.tsx`, `app/host/property/new/page.tsx`). Async components are not allowed inside client trees, so this would have required refactoring 7 unrelated pages to push their `"use client"` logic into child components. That violates CLAUDE.md's "no snuck-in refactors" rule and is out of scope for a homepage redesign. Switched to the loading-gated approach instead, which fixes the actual user-visible bug without dragging seven other files into the change.

## Risk notes

- **Photo hero asset is required.** `/photo-hero.png` must exist in `public/` or the homepage will render a broken image. Confirmed present after rename.
- **Logo swap is global.** Every page that uses `<Header />` now shows `logo-test.png`. If `logo-test.png` is a placeholder rather than the final brand asset, the user will see it everywhere, not just on the homepage.
- **Hero height is fixed at `h-[420px] md:h-[560px]`.** Tall enough to comfortably fit the title, subtitle, and search bar with room to breathe, but if the search bar grows in future (extra filters, etc) the bottom-aligned text could crowd the centre of the photo. Easy to bump if needed.
- **Skeleton-style fix means logged-out visitors still see "Become a Host" instantly** (correct), and hosts / admins see no link at all during the loading window (correct — they should never see it). Guests see it instantly too. The only behavioural change is hosts and admins no longer get the false-positive flash.
