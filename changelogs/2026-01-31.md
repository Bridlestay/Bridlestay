# Changelog - 2026-01-31

## Routes Create Mode - Toolbar Positioning Fix

### Issue
When clicking "Create Route", the toolbar (Plot, Erase, Insert, Snap, Undo, Clear, Style) was overlapping with the main navigation tabs at the top of the screen.

### Solution
Repositioned the `RouteCreatorToolbar` to appear **below** the main navigation bar, matching the OS Maps design pattern.

### Technical Changes

**File Modified:** `components/routes/route-creator.tsx`

| Property | Before | After |
|----------|--------|-------|
| `top` | `top-2` (8px) | `top-20` (80px) |
| `right` | `right-16` (64px) | `right-4` (16px) |
| `z-index` | `z-[1000]` | `z-30` |

**Before:**
```tsx
<div className="absolute top-2 right-16 z-[1000]">
```

**After:**
```tsx
<div className="absolute top-20 right-4 z-30">
```

### Why These Values
- `top-20` (80px): The nav tabs are at `top-4` (16px) and are approximately 60px tall. So 16 + 60 + small gap = ~80px positions the toolbar directly below them.
- `right-4` (16px): Aligns with the right edge of the navigation tabs which are also at `right-4`.
- `z-30`: Matches the z-index of the nav tabs. No need for `z-[1000]` which was excessive.

### Visual Result
- Main nav: Map | Saved Routes | Find Routes | Create Route (top row)
- Toolbar: Plot | Erase | Insert | Snap | Undo | Clear | Style (second row, below nav)
