# Changelog - 2026-01-31

## Routes Create Mode - Toolbar Positioning Fix

### Issue
When clicking "Create Route", the toolbar (Plot, Erase, Insert, Snap, Undo, Clear, Style) was overlapping with the main navigation tabs at the top of the screen.

### Solution
Repositioned the `RouteCreatorToolbar` to appear **below** the main navigation bar, matching the OS Maps design pattern.

### Technical Changes

**File Modified:** `components/routes/route-creator.tsx`

| Property | Before | After |
|----------|--------|-------|
| `top` | `top-2` (8px) | `top-20` (80px) |
| `right` | `right-16` (64px) | `right-4` (16px) |
| `z-index` | `z-[1000]` | `z-30` |

**Before:**
```tsx
<div className="absolute top-2 right-16 z-[1000]">
```

**After:**
```tsx
<div className="absolute top-20 right-4 z-30">
```

### Why These Values
- `top-20` (80px): The nav tabs are at `top-4` (16px) and are approximately 60px tall. So 16 + 60 + small gap = ~80px positions the toolbar directly below them.
- `right-4` (16px): Aligns with the right edge of the navigation tabs which are also at `right-4`.
- `z-30`: Matches the z-index of the nav tabs. No need for `z-[1000]` which was excessive.

### Visual Result
- Main nav: Map | Saved Routes | Find Routes | Create Route (top row)
- Toolbar: Plot | Erase | Insert | Snap | Undo | Clear | Style (second row, below nav)

## Create Mode - Multiple Fixes

### Issues Fixed

#### 1. "Start Recording" Button Removed from Create Mode & Desktop
- **Problem:** The "Start Recording" button was showing on desktop and in create mode
- **Solution:** 
  - Removed RouteRecorder entirely from create mode (it's for GPS recording while riding, not route plotting)
  - On main map view, wrapped RouteRecorder in `md:hidden` so it only shows on mobile

#### 2. Map Controls Positioning Fixed
- **Problem:** The map controls (layers, GPS, fullscreen, +/-) were positioned incorrectly on the left side with weird styling
- **Cause:** `className="left-[26rem]"` was conflicting with the default `right-4` positioning
- **Solution:** Removed the `left-[26rem]` className - controls now stay on the right side as intended

#### 3. Greyscale/Monochrome Toggle Now Works in Create Mode
- **Problem:** Toggling monochrome in the layer settings had no effect when creating a route
- **Cause:** The `monochrome` prop was not being passed to the map component in create mode
- **Solution:** Added `monochrome={layerSettings.monochrome}` to the RoutesMapV2 in create mode

### Technical Changes

**File Modified:** `app/routes/page.tsx`

| Change | Before | After |
|--------|--------|-------|
| RouteRecorder in create mode | Present | Removed |
| RouteRecorder on main view | Always visible | `md:hidden` (mobile only) |
| MapLayerControls className | `className="left-[26rem]"` | No className (uses default right positioning) |
| Map monochrome prop | Missing | `monochrome={layerSettings.monochrome}` |
