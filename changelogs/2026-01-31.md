## Routes Create Mode - Toolbar Positioning Fix

### Issue
When clicking "Create Route", the toolbar (Plot, Erase, Insert, Snap, Undo, Clear, Style) was overlapping with the main navigation tabs at the top of the screen.

### Solution
Repositioned the `RouteCreatorToolbar` to appear **below** the main navigation bar, matching the OS Maps design pattern.

### Technical Changes

**File Modified:** `components/routes/route-creator.tsx`

| Property | Before | After |
|----------|--------|-------|
| `top` | `top-2` (8px) | `top-20` (80px) |
| `right` | `right-16` (64px) | `right-4` (16px) |
| `z-index` | `z-[1000]` | `z-30` |

**Before:**
```tsx
<div className="absolute top-2 right-16 z-[1000]">
```

**After:**
```tsx
<div className="absolute top-20 right-4 z-30">
```

### Why These Values
- `top-20` (80px): The nav tabs are at `top-4` (16px) and are approximately 60px tall. So 16 + 60 + small gap = ~80px positions the toolbar directly below them.
- `right-4` (16px): Aligns with the right edge of the navigation tabs which are also at `right-4`.
- `z-30`: Matches the z-index of the nav tabs. No need for `z-[1000]` which was excessive.

### Visual Result
- Main nav: Map | Saved Routes | Find Routes | Create Route (top row)
- Toolbar: Plot | Erase | Insert | Snap | Undo | Clear | Style (second row, below nav)

## Route Preview Card - OS Maps Style with Thumbnail (v2)

### Overview
Updated the route preview card to match OS Maps design more closely, with a route thumbnail image on the left.

### New Layout
```
┌────────────────────────────────────────┐
│ Route Title                             │
│ ✓ Creator Name                          │
├────────────────────────────────────────┤
│ ┌──────┐   🐴 12.5 km                   │
│ │ MAP  │   ⏱️ 1 h 30 min                │
│ │THUMB │   [Moderate]                   │
│ └──────┘                                │
├────────────────────────────────────────┤
│          View details                   │
└────────────────────────────────────────┘
```

### Features Added
1. **Route Thumbnail** - Uses Google Static Maps API to generate a 90x90px preview
   - Shows actual route path on terrain map
   - Path simplified to ~30 points to stay under URL limits
   - Falls back to placeholder SVG if generation fails

2. **Verified checkmark** - Green checkmark SVG next to creator name

3. **Clean stats layout** - Distance and time stacked vertically next to thumbnail

4. **Subtle hover effect** - View details link changes to light green on hover

### Technical Implementation
- Static map URL: `https://maps.googleapis.com/maps/api/staticmap?...&path=color:0x4338CA|weight:3|{coords}`
- Path coordinates sampled every Nth point to keep URL under 8KB limit
- Uses `process.env.NEXT_PUBLIC_GOOGLE_MAPS_API_KEY` for API access
