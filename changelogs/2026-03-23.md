# Changelog — 2026-03-23

## Session 1

### Summary
Route visibility overhaul — card-style UI selector (matching Padoq design language), visibility badges in saved routes, share-token enforcement for "Anyone with Link" routes, and fixed all app code references to the dropped `reviews` table.

### Changes

#### Visibility selector UI redesign
- **Files modified**: `components/routes/save-route-modal.tsx`, `components/routes/route-creator.tsx`
- **What changed**: Replaced vertical radio button list with a 3-column card grid selector matching the reference design. Each option shows a centered icon, label, and description. Selected card has green (#267347) border with subtle background tint. Removed unused RadioGroup/RadioGroupItem imports.

#### Visibility badges in saved routes
- **File modified**: `components/routes/saved-routes-panel.tsx`
- **What changed**: Added visibility badges next to the difficulty badge in the "My Routes" tab. Shows "Link Only" (blue) or "Private" (gray) badges with corresponding icons. Public routes don't show a badge (it's the default state — no badge needed).

#### Share token API endpoint
- **File created**: `app/api/routes/shared/route.ts`
- **What it does**: GET endpoint that fetches a route by `share_token` query parameter. Uses service client to bypass RLS. Only returns routes with `visibility = 'link'`. This enables the "Anyone with Link" feature — shared URLs use `/api/routes/shared?token=xyz`.

#### Migration 084: Fix link visibility RLS
- **File created**: `supabase/migrations/084_fix_link_visibility_rls.sql`
- **What it does**:
  - Recreates the `routes_select` RLS policy (unchanged from 081 — link-only routes are accessed via service client)
  - Ensures the `generate_route_share_token()` trigger function exists and fires on INSERT/UPDATE, auto-generating a 32-char hex token when `visibility = 'link'`

- `supabase/migrations/084_fix_link_visibility_rls.sql`

### Files Modified

## Session 2

### Summary
UI polish and UX improvements — taller visibility cards with green checkmark tick, elevation graph label de-overlapping and edge clamping, waypoint list timeline redesign with collapsible behaviour, waypoint map-click now scrolls to waypoint in panel instead of showing popup, improved distance feedback for waypoint placement.

### Changes

#### Visibility card polish
- **Files modified**: `components/routes/save-route-modal.tsx`, `components/routes/route-creator.tsx`
- **What changed**: Increased card vertical padding (`p-3` → `px-3 py-4`), added `relative` positioning, added green checkmark circle (white check icon in `#267347` circle) in top-right corner when card is selected. Added `Check` import from lucide-react.

#### Elevation graph key point labels
- **File modified**: `components/routes/elevation-profile.tsx`
- **What changed**:
  - Removed `overflow-hidden` from outer container so labels at edges aren't clipped
  - Added `overflow-visible` to chart area div
  - Added de-overlap logic: key points closer than 12% apart are spread symmetrically, with alternating above/below label positions
  - Added edge clamping: labels near 0% left-align, labels near 100% right-align (instead of always centering) to prevent text running off-screen

#### Waypoint list timeline redesign
- **File modified**: `components/routes/route-waypoints-panel.tsx`
- **What changed**:
  - Replaced flat list with vertical timeline layout: thick dotted line (3px `border-dotted`) connecting numbered circles on the left
  - Nicer numbered circles: larger (36px), bolder, with ring highlights for Start (green + ring) and Finish (red + ring), dark slate for numbered waypoints
  - Collapsible behaviour: if >4 waypoints, only first 3 + last (finish) shown by default, with "Show X more waypoints" expand button in the timeline gap
  - "Show fewer waypoints" collapse button when expanded
  - Added `showAllWaypoints` state and `COLLAPSED_LIMIT = 4` constant

#### Waypoint map click → scroll to list
- **Files modified**: `components/routes/routes-map-mapbox.tsx`, `app/routes/page.tsx`, `components/routes/route-waypoints-panel.tsx`
- **What changed**:
  - Map: popup dismissed on waypoint click (was persisting on mobile)
  - Page: `handleWaypointClick` no longer early-returns in waypoints view mode — always triggers scroll. Uses clear-then-set pattern to force re-trigger for same waypoint
  - Panel: added `useEffect` that auto-scrolls to `initialExpandedId` with `scrollIntoView({ behavior: "smooth", block: "center" })`, expands the target waypoint, shows all waypoints if collapsed, and adds a brief ring highlight animation (1.5s)

#### Waypoint placement distance feedback
- **File modified**: `components/routes/routes-map-mapbox.tsx`
- **What changed**: Toast message for "too far from route line" now includes actual distance and tolerance in metres (e.g. "Too far from the route line (150m away). Click within 50m of the route to place a waypoint.")

### Files Modified
