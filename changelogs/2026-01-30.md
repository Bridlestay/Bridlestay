# Changelog - 2026-01-30

## Major Routes System Redesign - OS Maps Style

Today marks a major overhaul of the routes system to create an experience similar to OS Maps. The goal is to have a map-first, app-like interface where the map takes center stage.

---

## 1. New Map-First Layout

**What Changed:**
The routes page now features a full-screen map (100% viewport height) instead of the previous card-based layout with a map in a small container.

**Before:** 
- Map in a `h-96` card (fixed 384px height)
- Route list below/beside the map
- Traditional web page layout with header and container padding

**After:**
- Map fills the entire screen (`fixed inset-0`)
- Overlay panels float on top of the map
- App-like experience similar to OS Maps

## 9. GPS-Based Hazard Placement Restriction (NEW)

**What Changed:**
Hazard reports now require the user to be near the route. This prevents spam/false hazard reports from users who aren't actually on the route.

**Implementation:**
- When user opens the hazard dialog, their GPS location is checked
- User must be within **500 meters** of the route to submit a hazard
- Shows status banner:
  - "Checking your location..." (with spinner)
  - "Location verified - you're near the route" (green, can submit)
  - "Too far from route" (orange, shows distance, retry button)
  - "Location Error" (red, shows error message, retry button)
- **Admins and route owners** can bypass the location check
- Location is calculated using Haversine formula

**Files Modified:**
- `components/routes/route-detail-drawer.tsx` - Added location checking logic and UI

## 10. Collapsible Discussion Preview (NEW)

**What Changed:**
The Discussion (comments) tab now has a collapsed preview mode with horizontal scrolling.

**Implementation:**
- **Collapsed View (Default):**
  - Shows "X Comments" header
  - Horizontal scroll carousel of top 5 comment preview cards
  - Each card shows avatar, name, truncated comment (2 lines), timestamp, reply count
  - "View All X Comments" expand button
- **Expanded View:**
  - Full comment input form
  - Full Instagram-style comment thread with replies
  - "Collapse Discussion" button at top

**Why:**
- Reduces visual clutter in the route detail panel
- Allows quick preview without loading full discussion
- Better mobile experience with horizontal swiping
