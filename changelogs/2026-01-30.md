# Changelog - 2026-01-30

## Major Routes System Redesign - OS Maps Style

Today marks a major overhaul of the routes system to create an experience similar to OS Maps. The goal is to have a map-first, app-like interface where the map takes center stage.

---

## 1. New Map-First Layout

**What Changed:**
The routes page now features a full-screen map (100% viewport height) instead of the previous card-based layout with a map in a small container.

**Before:** 
- Map in a `h-96` card (fixed 384px height)
- Route list below/beside the map
- Traditional web page layout with header and container padding

**After:**
- Map fills the entire screen (`fixed inset-0`)
- Overlay panels float on top of the map
- App-like experience similar to OS Maps

**Files Modified:**
- `app/routes/page.tsx` - Complete rewrite with new layout

---

## 2. Custom Map Controls (OS Maps Style)

**What Changed:**
Removed all default Google Maps UI controls and replaced with custom controls positioned bottom-right:

**New Controls:**
1. **Layer Toggle Button** - Opens the layer settings panel
2. **GPS Location Button** - Centers map on user's location
3. **Fullscreen Button** - Toggles fullscreen mode
4. **Zoom Controls** - Plus/minus buttons in a pill shape

**Removed:**
- Google Maps type selector
- Google Maps zoom controls
- Google Maps fullscreen button

**Files Created:**
- `components/routes/map-layer-controls.tsx` - New custom controls component

**Files Modified:**
- `components/routes/routes-map-v2.tsx` - Disabled Google's default UI

---

## 3. Layer Settings Panel

**What Changed:**
Created a slide-up panel (mobile) / side panel (desktop) for map layer settings.

**Features:**
- **Map Type Selection** - Visual buttons for Standard, Topographic, Aerial
- **Path Layer Toggles:**
  - Bridleways (brown) - ON by default
  - Byways/BOATS (purple) - ON by default
  - Restricted Byways (orange) - ON by default
  - Footpaths (green) - OFF by default (hidden to discourage hikers)
- **Feature Toggles:**
  - Waymarkers visibility
  - Hazards visibility
  - Properties visibility
- **Route Line Width** - Slider to adjust route line thickness (2-8px)

**Files Created:**
- `components/routes/map-layer-controls.tsx`

---

## 4. Search Panel (OS Maps Style)

**What Changed:**
Created a collapsible search panel that slides in from the left side.

**Features:**
- **Toggle Button** - Hamburger menu when closed, chevron when open
- **Search Bar** - Search for places and routes
- **Tabs** - Places / Routes (like OS Maps)
- **Filters Panel:**
  - Distance range slider (0-50+ miles)
  - Difficulty buttons (Unrated, Easy, Moderate, Difficult, Severe)
  - Minimum rating filter (Any, 3+, 3.5+, 4+, 4.5+)
  - Route type filter (Circular, Point-to-point)
- **Routes List** - Scrollable list with cover photos, stats, badges
- **Create Route Button** - At the bottom of the panel

**Files Created:**
- `components/routes/map-search-panel.tsx`

---

## 5. Route Bottom Sheet

**What Changed:**
When clicking a route on the map, instead of opening a drawer immediately, a bottom sheet appears with route preview info.

**Features:**
- **Single Route View:**
  - Cover image, title, creator info
  - Distance, duration, rating
  - Difficulty and route type badges
  - Close button
  - Tap to open full details
  
- **Cluster/Multiple Routes View (Carousel):**
  - Horizontal scrollable carousel of route cards
  - Each card shows thumbnail, title, stats
  - Tap to highlight route on map
  - Double-tap to open full details
  - Navigation arrows on desktop

**Files Created:**
- `components/routes/route-bottom-sheet.tsx`

---

## 6. Footpaths Hidden by Default

**What Changed:**
As requested, footpaths are now hidden by default to discourage walkers/hikers and reduce API usage.

**Implementation:**
- `showFootpaths` defaults to `false` in LayerSettings
- Toggle is still available in the layer panel but styled with reduced opacity
- Labeled as "(Hidden)" to indicate it's off by default
