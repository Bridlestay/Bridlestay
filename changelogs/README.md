# Changelogs

This folder contains dated changelog files documenting all changes made to the Bridlestay/Cantra codebase.

## Format

Each changelog file is named `YYYY-MM-DD.md` and contains:

- **Session summary** - Brief overview of what was accomplished
- **Changes made** - Detailed list of each change with:
  - Files created/modified/deleted
  - What was changed and WHY
  - Any risks or notes
- **Verification steps** - How to test the changes
- **Open questions** - Any unresolved items

## Usage

- Review changelogs before starting work to understand recent changes
- Reference changelogs when debugging to understand when/why something changed
- Use for onboarding to understand project evolution
