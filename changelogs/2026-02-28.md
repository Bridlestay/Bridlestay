### 3. Code Review Fixes

Issues found during systematic review of recent changes:

**Fix: Photos route storage client (security hardening)**
**File:** `app/api/routes/[id]/photos/route.ts`

Reverted storage upload from service client back to regular authenticated client. The storage bucket policy already allows authenticated uploads (`auth.role() = 'authenticated'`). The service client should only be used for database inserts where RLS blocks the operation. This follows the principle of least privilege — storage operations use the user's auth context, only DB operations bypass RLS.

**Fix: Normalize photo timestamps in completions API**
**File:** `app/api/routes/[id]/completions/route.ts`

Photos from `route_user_photos` have an `uploaded_at` column while `route_photos` has `created_at`. When merging photos from both tables, the timestamp field names were inconsistent. Fixed by mapping `uploaded_at` → `created_at` on user photos so all photos have a uniform `created_at` field. Also added time-based sorting so photos from both sources are properly ordered (newest first).

### Key technical notes for the next session

- **Two photo tables**: `route_user_photos` (community/review photos, column: `uploaded_at`) and `route_photos` (owner/admin photos, column: `created_at`). Different timestamp column names — always check which table you're querying.
- **PostgREST silent failures**: Querying a non-existent column returns `{ data: null, error: {...} }`, not a thrown exception. Combined with `data || []` fallback, this silently hides all results. Be careful with column names.
- **Service client usage**: `createServiceClient()` (from `lib/supabase/service.ts`) bypasses RLS. Use ONLY for DB operations where RLS blocks (e.g., cross-user queries). Storage operations should use the regular authenticated client — the bucket policy allows `auth.role() = 'authenticated'`.
- **Route geometry**: Stored as GeoJSON LineString `{ type: "LineString", coordinates: [[lng, lat], ...] }`. The `route` object may have it as `geometry` or `route_geometry` depending on context.
