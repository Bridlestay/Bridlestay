### 2. Next.js Routing Conflict Fix

**Why:** Server wouldn't start due to conflicting dynamic route parameter names.

**Problem:** Two folders under `app/api/properties/`:
- `[id]/` - used by share endpoint
- `[propertyId]/` - used by blocked-dates and nearby-routes

Next.js requires consistent naming at the same path level.

**Solution:** Standardized on `[id]`

**Files created:**
- `app/api/properties/[id]/blocked-dates/route.ts`
- `app/api/properties/[id]/nearby-routes/route.ts`

**Files deleted:**
- `app/api/properties/[propertyId]/blocked-dates/route.ts`
- `app/api/properties/[propertyId]/nearby-routes/route.ts`
- `app/api/properties/[propertyId]/` (entire folder)

**Code changes:** Updated parameter types from `{ params: { propertyId: string } }` to `{ params: Promise<{ id: string }> }` (Next.js 14 pattern).
