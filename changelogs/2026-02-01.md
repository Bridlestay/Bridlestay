## 2. Edit Route - Zoom Level Fix

### Issue
When clicking "Edit Route" from the route details panel, the map would zoom all the way out instead of maintaining the current view or zooming to fit the route.

### Root Cause
The `fitBounds` function was being called with an incorrect format. The edit page was passing a bounds object, but the function expected `[lng, lat][]` coordinates.

### Solution
Fixed the edit route page to pass coordinates in the correct format:

**Before:**
```tsx
const paddedBounds = {
  north: bounds.north + latPadding,
  south: bounds.south - latPadding,
  // ...
};
mapRef.current.fitBounds?.(paddedBounds);
```

**After:**
```tsx
const coordinates: [number, number][] = waypoints.map(wp => [wp.lng, wp.lat]);
mapRef.current.fitBounds?.(coordinates);
```

### Files Modified

**`app/routes/edit/[id]/page.tsx`**
- Simplified the `fitBounds` call to use the correct coordinate format

**`components/routes/routes-map-v2.tsx`**
- Updated `fitBounds` interface to accept optional padding parameter
- Reduced default left padding from 420px to 100px for edit mode

## 3. Edit Route - Waypoints Now Load Correctly

### Verification
The edit route functionality was verified to be working correctly:
- Route geometry coordinates are extracted and converted to waypoints
- Waypoints are displayed on the map as editable markers
- Users can add, remove, and drag waypoints
- The route type (linear/circular) is preserved

### How It Works
1. Route data is fetched from `/api/routes/{id}`
2. Geometry coordinates are extracted: `routeData.geometry.coordinates`
3. Each coordinate `[lng, lat]` is converted to a Waypoint object
4. Waypoints are set in state and rendered on the map
5. Map zooms to fit all waypoints
