# Changelog — 2026-02-24

## Session: UX Polish, Hazard Fixes, UI Design Guide

Continued from the 2026-02-22 session (community warnings + waypoint tags). This session focused on UX issues found during testing and establishing a formal UI/UX design guide.

---

### Bug Fixes

#### 1. Clickable Hazards Badge
The red hazards count badge at the top of the route card was not clickable (unlike the warnings badge). Now clicking it opens the hazards map view mode, or scrolls to the hazards section if the map view isn't available.

#### 2. Hazard Popup Persistence
Hazard map popups were dismissed instantly on mouse leave, making it impossible to click the "Mark as Cleared" button. Added a 200ms close delay with hover detection on the popup itself — users can now move their mouse from the marker to the popup to interact with it.

#### 3. Hazards Default Off in Map Settings
`showHazards` was defaulting to `true` in layer settings. Changed to `false` so hazard markers don't show until explicitly toggled on.

#### 4. Layer Settings Toggle Works for Hazards
Hazard markers were only shown in view mode (`mapViewMode === "hazards"`) and ignored the layer settings toggle. Now they respond to both, matching the existing pattern for waypoints: `mapViewMode === "hazards" || layerSettings.showHazards`.

#### 5. Back Button Visible on Mobile
The "Back to route details" button in view mode (waypoints/hazards) had `z-30` but the mobile header has `z-50`, making it hidden. Raised to `z-[60]` and repositioned to `top-14` so it sits below the mobile header and is always clickable.

#### 6. Hazards Toggle Syncs with View Mode
Clicking the hazards badge on the route card enters hazards view mode (`mapViewMode = "hazards"`), which shows hazard markers on the map. However, the hazards toggle in map settings didn't reflect this — it stayed off. Now `handleEnterViewMode("hazards")` also sets `layerSettings.showHazards = true`, so the toggle is in sync.

### Risk Notes

- The z-[60] on the back button exceeds the standard z-50 header. This is intentional — the back button must be above the header when in view mode. When view mode is exited, the button is removed entirely.
- `showHazards: false` default means existing users who had hazards visible will need to re-enable via the toggle. This is the correct UX — hazards should be opt-in.
