# Changelog — 2026-02-26

## Session: Review Edit Pre-fill, Compact Review Cards, Map Pin Auto-Refresh

Three changes: review editing now pre-fills existing data, review cards redesigned to a compact expandable format inspired by OS Maps, and map route pins auto-refresh every 10 seconds when on the main map.

---

### 1. Review Edit Pre-Population

**File:** `components/routes/route-detail-drawer.tsx`

When clicking "Update my review", the review flow now pre-fills:
- **Rating** (step 1) — stars pre-selected from existing `completion.rating`
- **Tags** (step 3) — checkboxes pre-ticked from existing `completion.tags`
- **Short note** (step 4) — textarea pre-filled from existing `completion.short_note`
- **Step navigation** — `maxVisitedStep` set to 4 so all steps are clickable, allowing the user to jump directly to whichever part they want to edit or skip past unchanged steps

Previously, clicking "Update my review" started fresh every time, losing all previous review data.

**Fix for stale data:** The completions API now also merges `rating` from `route_reviews` as a fallback (since bare completions from the old `/review` endpoint have no rating). The short note pre-fill falls back to `review_body` when `short_note` is empty (covers old reviews where notes weren't stored in the completion).

### 2. Compact Review Cards (OS Maps Inspired)

**File:** `components/routes/route-detail-drawer.tsx`

Redesigned review cards from large full-detail cards to a compact expandable format:

**Collapsed state (default):**
- Compact single row: avatar (32px), name + date on same line, 2-line text clamp
- Photo teaser strip: small 48x64px thumbnails previewing photos (up to 3 + overflow count)
- "Tap to read more" hint when there's more content (long text, photos, or tags)
- Clean border, no background fill — lightweight and scannable

**Expanded state (tap to open):**
- Full review text with `whitespace-pre-line` for line breaks
- Tags as pill-shaped spans with emoji
- Full photo grid (responsive 1/2/3 columns based on count)
- Photo click opens lightbox (with `e.stopPropagation()` to prevent collapse)
- "Show less" collapse button at bottom

**Other changes:**
- Default visible reviews increased from 3 to 5 (compact cards take less space)
- Section header simplified: "Rider Reviews" with count text
- "Show all" button as subtle outline pill

### 3. Map Pin Auto-Refresh

**File:** `app/routes/page.tsx`

Route pins on the main map now auto-refresh every 10 seconds. This ensures pins appear even if the initial load had a timing issue or if new routes have been published.

**Conditions** — polling only runs when:
- `activeTab === "map"` (user is on the map tab)
- `!isCreating` (not in route creation mode)
- `!selectedRouteId` (no route detail drawer open)

The interval is cleaned up when any condition changes (user selects a route, switches tab, starts creating). This avoids unnecessary API calls in non-map states.

### 4. Completions API Resilience (from prior session, included in this push)

**File:** `app/api/routes/[id]/completions/route.ts`

- Graceful fallback when `review_text` column doesn't exist (tries `review_text` first, falls back to `body`)
- Photos included per completion from both `route_user_photos` and `route_photos` tables
- Each completion response now includes `{ photos: [...] }` array

### 5. Review Update Actually Saves Tags & Notes (Complete Endpoint Upsert)

**File:** `app/api/routes/[id]/complete/route.ts`

The `/complete` endpoint previously only supported INSERT. When updating a review, it failed with 409 (unique constraint) and the updated tags/short_note were silently lost. The old completion's empty `notes` field was preserved.

**Fix:** Changed to check-then-upsert pattern:
- Queries for existing completion by `user_id + route_id`
- If found: UPDATE the existing row's `notes` and `rating` fields (preserves `completed_at`)
- If not found: INSERT a new row as before

This means editing a review now correctly saves updated tags and short notes to the completion.

### 6. Long Note Pre-Population & Preservation

**File:** `components/routes/route-detail-drawer.tsx`

When editing a review that previously had a long note (step 5):
- The long note is extracted from `review_body` by splitting on the `\n\n---\n\n` separator
- `reviewLongNote` is pre-populated so step 5 shows the existing extended text
- `maxVisitedStep` set to 5 (was 4) so step 5 is navigable during edits
- `handleSubmitRideReview` now includes the long note in `review_text` if present, preventing the update from overwriting combined text with just the short note

### 7. Clickable Reviewer Profiles

**File:** `components/routes/route-detail-drawer.tsx`

Reviewer avatars and names in the compact review cards are now clickable links to `/profile/[userId]`. The avatar shows a green ring on hover as a visual affordance. Uses `e.stopPropagation()` to prevent the click from toggling the card expand/collapse.

### Risk Notes

- Auto-refresh makes 2 API calls (search + my-routes) every 10 seconds. Both are simple Supabase queries. For heavy traffic this could be optimised with cache headers or conditional fetching, but for current scale this is fine.
- The `/complete` endpoint now does a SELECT + UPDATE/INSERT instead of a single INSERT. This adds one extra query but is necessary for correct update behavior.
- Compact card photo teasers use small fixed-size thumbnails (48x64px). These load full images scaled down — could be optimised with image transforms in the future.
