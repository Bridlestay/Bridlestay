# Changelog — 2026-03-13

## Session 1: Fix route edit saving as new route + no-photo rounded corners

### Summary
Fixed two issues: (1) Editing a route created a new route instead of updating because the SaveRouteModal kept stale form state between uses, and (2) the "No photos yet" placeholder in the route detail modal had no visible rounded corners.

### Files Modified
- `app/routes/page.tsx` — Added `key` prop to `SaveRouteModal` to force remount between create/edit modes
- `components/routes/route-detail-drawer.tsx` — Inset the "No photos yet" placeholder with padding and rounded corners

### What Changed and Why

**Route edit creating new routes (Issue 1)**:
- **Root cause**: The `SaveRouteModal` component uses `useState` to initialize form fields from `existingData` props. However, `useState` initializers only run on first mount. Since the modal stays mounted (returns `null` when closed rather than being removed from the tree), switching between create and edit modes left stale form state — empty fields, wrong button text, and incorrect save behavior.
- **Fix**: Added a `key` prop (`key={isEditing ? 'edit-${editingRouteId}' : 'create'}`) to the `SaveRouteModal`. When the key changes (switching between create/edit or editing a different route), React unmounts and remounts the component, ensuring `useState` initializers run fresh with the correct `existingData`. This means the form pre-populates with existing title, description, visibility, and difficulty when editing, and the button correctly shows "Update Route".

**No-photo rounded corners (Issue 2)**:
- **Root cause**: The "No photos yet" green gradient filled the entire photo hero area edge-to-edge. While the card's `overflow-hidden rounded-2xl` technically clipped the corners, the light green gradient against the white card made the rounding visually invisible.
- **Fix**: Wrapped the green gradient in a `p-4` padding container and added `rounded-xl` to the gradient div. This insets the placeholder from the card edges, showing the white card background at the corners and giving the green area its own visible rounded corners.

### Verification Steps
1. `npm run build` passes
2. **Edit existing route**: Open route detail → three-dot menu → Edit → modify route → Save Route → modal shows "Update Route" button with pre-filled title/description → saves correctly as update (no new route created)
3. **Create new route**: Start fresh route → Save Route → modal shows "Save Route" button with empty fields → creates new route
4. **No-photo styling**: Open a route with no photos → the "No photos yet" area has visible rounded corners with white card background at the edges
5. **Photo styling**: Routes with photos still show edge-to-edge photo carousel (no change)
