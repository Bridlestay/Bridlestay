# Changelog — 2026-03-16

## Session 1 (Morning)

### Summary
Major navigation UI overhaul — Google Maps-inspired bottom card with ETA/remaining distance, drag-up actions menu with "Add a Report" for hazards, 45° tilted map perspective, thicker outlined route line, hidden S/E markers during navigation.

### Changes

#### Redesign: Bottom card with ETA (Google Maps style)
- **File modified**: `components/routes/route-navigator.tsx`
- **What changed**: Complete rewrite of bottom navigation panel. Now shows estimated time remaining (green, large — e.g. "20 min"), distance remaining, and ETA clock time (e.g. "2.1 km · 14:46") instead of elapsed time/distance. Elapsed stats tracked silently for post-ride summary. Bottom card is full-width with rounded top corners and centered drag handle pill. No scrollbar.
- **Why**: Previous version showed elapsed time counting up, which isn't useful during riding. Google Maps shows time remaining + ETA — what riders actually need.

#### Feature: Drag-up actions menu
- **File modified**: `components/routes/route-navigator.tsx`
- **What changed**: Dragging up on the bottom card handle reveals 3 actions: Add a Report, Sound on/off, Navigation Settings. Touch gesture detection via `touchStart`/`touchEnd` with 40px threshold. Click on handle also toggles.
- **Why**: Keeps the main card minimal (stats + pause + exit) while providing access to actions. Matches Google Maps swipe-up pattern.

#### Feature: Add a Report (quick hazard picker)
- **File modified**: `components/routes/route-navigator.tsx`
- **What changed**: "Add a Report" opens a bottom sheet with a 3-column grid of hazard types (tree fall, flooding, livestock, closure, visibility, ice, overgrown, damaged path, crossing, erosion, gate locked, other). Tap to select, optional text field for details, submit. Auto-captures rider's current GPS location. Posts to existing `/api/routes/[id]/hazards` endpoint.
- **Why**: Riders need to report hazards they encounter while riding. Quick picker minimizes friction — one tap for the type, submit. Can't type much on a horse.

#### Feature: 45° map tilt in follow mode
- **File modified**: `components/routes/routes-map-mapbox.tsx`
- **What changed**: When `followUser` is true, `easeTo` now includes `pitch: 45` and `zoom: Math.max(current, 17)` for a "road ahead" perspective. Pitch resets to 0° when follow mode is exited.
- **Why**: Google Maps tilts during navigation for better spatial awareness of the path ahead. Flat top-down view doesn't convey direction of travel well.

#### Feature: Thicker outlined route line during navigation
- **File modified**: `components/routes/routes-map-mapbox.tsx`
- **What changed**: When `followUser` is true, adds a dark blue outline layer (12px, #1E40AF) behind the main route line (8px, #3B82F6, full opacity). Outline layer is added via `map.addLayer()` with `before: "routes-line"`. Removed and restored to normal styling when navigation ends.
- **Why**: The default route line is too thin to follow during navigation, especially on a tilted map. Google Maps uses a thick outlined line.

#### Feature: Hidden S/E markers during navigation
- **File modified**: `components/routes/routes-map-mapbox.tsx`
- **What changed**: When `followUser` is true, S/E markers are hidden via `el.style.display = "none"`. Shown again when follow mode ends. Prevents the large green S and red E circles from blocking the view during active navigation.
- **Why**: S/E pins are 28px circles that sit right on the route. During navigation, the rider's arrow and route line are sufficient — the pins just clutter the view.

#### Improvement: ETA based on actual riding speed
- **File modified**: `components/routes/route-navigator.tsx`
- **What changed**: ETA calculated from rider's actual average speed applied to remaining distance. Kicks in after 200m traveled (before that, shows total distance only). Adapts to actual pace — walking, trotting, cantering. Shows clock time ETA (e.g. "14:46").
- **Why**: Fixed speed assumptions don't match reality. Actual speed adapts to the rider's pace.

#### Cleanup: Removed broken audio alert
- **File modified**: `components/routes/route-navigator.tsx`
- **What changed**: Removed `<audio>` element referencing non-existent `/sounds/alert.mp3`. Voice announcements via `speechSynthesis` API still work and are controlled by the sound toggle.
- **Why**: The alert sound file never existed — the reference was dead code.

### Files Modified
- `components/routes/route-navigator.tsx` — Complete rewrite: ETA card, drag-up actions, report picker, paused indicator
- `components/routes/routes-map-mapbox.tsx` — 45° pitch, thick outlined route line, hidden S/E markers during nav

4. Drag up on the handle → actions list appears: Add a Report, Sound, Settings
5. Tap "Add a Report" → grid of hazard icons appears, select one, optional details, submit
6. Map should tilt to ~45° and zoom to 17 during navigation
7. Route line should be thicker with a dark blue outline

### Risk Notes
- ETA accuracy depends on sufficient travel distance (200m minimum). Before that, only shows route distance
- Report submission requires auth — unauthenticated riders will get a 401
- Route outline layer is added dynamically — if the map style changes mid-navigation, the layer could be lost. The `styleLoadCount` dependency isn't tracked here but unlikely to change during navigation
- 45° pitch may feel unfamiliar on first use — riders accustomed to flat maps might find it disorienting initially

## Session 2 (Afternoon)

### Summary
Navigation polish fixes — removed plant emoji, fixed S/E marker visibility timing, fixed sound toggle collapsing drawer, added smooth CSS animation for drawer expand/collapse, hazard report now shows toast when GPS not ready, completion threshold tightened to 20m, Navigate button hidden on desktop.

### Changes

#### Fix: Remove plant emoji from bottom card
- **File modified**: `components/routes/route-navigator.tsx`
- **What changed**: Removed the 🌿 emoji that appeared next to the green ETA number.

#### Fix: Sound toggle no longer collapses drawer
- **File modified**: `components/routes/route-navigator.tsx`
- **What changed**: Removed `setDrawerExpanded(false)` from the Sound on/off button click handler. Sound toggles in place while the drawer stays open.

#### Fix: Smooth drawer expand/collapse animation
- **File modified**: `components/routes/route-navigator.tsx`
- **What changed**: Replaced conditional render (`{drawerExpanded && ...}`) with CSS `max-height` transition. `overflow-hidden transition-all duration-300 ease-in-out` with `maxHeight: drawerExpanded ? "200px" : "0px"`. Actions section smoothly slides in/out.

#### Fix: Hazard report shows GPS feedback
- **File modified**: `components/routes/route-navigator.tsx`
- **What changed**: When GPS position isn't available yet, tapping Submit Report now shows toast "Waiting for GPS location. Please try again in a moment." instead of silently doing nothing.

#### Fix: S/E markers properly hidden during navigation
- **File modified**: `components/routes/routes-map-mapbox.tsx`
- **What changed**: S/E markers now have `display: none` set at creation time when `followUser` is true. Previously, markers were created visible and only hidden by a separate effect that fired later — causing a flash of visible markers. Also added `followUser` to the dependency array so markers are recreated when navigation starts/stops.

#### Change: Completion threshold 30m → 20m
- **File modified**: `components/routes/route-navigator.tsx`
- **What changed**: Route completion now triggers when rider is within 20m of endpoint (was 30m). Tighter threshold feels more accurate while still accounting for GPS drift in rural areas.

#### Feature: Navigate button hidden on desktop
- **File modified**: `components/routes/route-detail-drawer.tsx`
- **What changed**: Added `md:hidden` to the Navigate button. Desktop users can still view routes but cannot start GPS navigation — it only makes sense on mobile.

### Files Modified
- `components/routes/route-navigator.tsx` — Plant emoji, sound toggle fix, drawer animation, GPS toast, 20m threshold
- `components/routes/routes-map-mapbox.tsx` — S/E marker hide timing fix
- `components/routes/route-detail-drawer.tsx` — Navigate button hidden on desktop
