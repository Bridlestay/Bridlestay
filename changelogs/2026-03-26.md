# Changelog — 2026-03-26

## Session 1

### Summary
Fixed 6 UI/UX issues: compact visibility cards (removed oversized aspect ratio), desktop locate-me button hidden, waypoint timeline gradient fade-out and expand animation fixed, waypoint panel indentation corrected, and waypoint/hazard markers now hide when zoomed out to prevent overlap with route clusters.

### Changes

#### Visibility cards: compact portrait sizing
- **Files modified**: `components/routes/save-route-modal.tsx`, `components/routes/route-creator.tsx`
- **What changed**:
  - Removed `aspect-[3/4]` which made cards far too large
  - Switched from `grid-cols-3 gap-2` to `flex justify-center gap-3` for tighter centering
  - Fixed width `w-[90px]` with `py-5` padding for natural portrait proportion
  - Icons increased to `h-7 w-7`, label text `text-xs font-semibold`
  - Shorter descriptions ("Visible to everyone", "Shared via link", "Only you can see")
  - Smaller checkmark badge: `w-4 h-4`

#### Locate-me button: hidden on desktop
- **File modified**: `components/routes/map-layer-controls.tsx`
- **What changed**: Wrapped locate-me button in `<div className="md:hidden">` so it only appears on mobile devices. Desktop users don't need GPS location on a desktop browser.

#### Waypoint timeline: gradient fade-out fixed
- **File modified**: `components/routes/waypoint-timeline.tsx`
- **What changed**:
  - Increased gradient overlay height from 16 to 20 (`-mt-20 pt-20`)
  - Added `via-white/80` mid-stop for a stronger, more visible fade effect
  - Extended gradient bleed with `inset-x-[-10px]` to cover the timeline dotted line
  - Bumped z-index to `z-20` to sit above waypoint cards

#### Waypoint timeline: expand/collapse animation fixed
- **File modified**: `components/routes/waypoint-timeline.tsx`
- **What changed**:
  - Replaced unreliable `animate-in` CSS classes with `max-h` + `opacity` CSS transitions
  - Hidden content mounts at `max-h-0 opacity-0`, transitions to `max-h-[5000px] opacity-100`
  - Collapse triggers CSS transition first, then unmounts after 300ms delay
  - Uses `requestAnimationFrame` to ensure DOM paints before triggering the transition

#### Waypoint panel: indentation fix
- **File modified**: `components/routes/route-waypoints-panel.tsx`
- **What changed**: Fixed misaligned indentation of the expanded details section and its closing tags to properly nest within the content container.

#### Clustering: waypoint markers hidden when zoomed out
- **File modified**: `components/routes/routes-map-mapbox.tsx`
- **What changed**:
  - Added `zoom` event listener that sets `display: none` on waypoint and hazard HTML markers when zoom drops below level 11
  - Markers reappear when zooming back in past the threshold
  - Runs on mount to set correct initial visibility
  - Prevents waypoint markers from overlapping with route cluster circles
