## 5. UK-Only Route Creation

### Overview
Restricted route creation to UK & Ireland bounds only.

### Changes
- Added UK bounds constant: N 60.86°, S 49.86°, E 1.77°, W -8.65°
- Added `isWithinUK()` validation function
- When user clicks outside UK during route creation, shows toast error
- Message: "Routes can only be created within the UK & Ireland"

### Technical Implementation
```typescript
const UK_BOUNDS = {
  north: 60.86,  // Shetland
  south: 49.86,  // Channel Islands
  east: 1.77,    // Lowestoft
  west: -8.65,   // Western Ireland/Scotland
};

function isWithinUK(lng: number, lat: number): boolean {
  return lat >= UK_BOUNDS.south && 
         lat <= UK_BOUNDS.north && 
         lng >= UK_BOUNDS.west && 
         lng <= UK_BOUNDS.east;
}
```

## 8. Fixed Mobile Route View Pipeline

### Overview
Fixed issue where clicking "Map" button when viewing a route from Find Routes would incorrectly return to Find Routes instead of showing the route on the map.

### The Problem
When opening a route from "Find Routes" and then clicking the "Map" button to view the route on the map, the user was taken back to the Find Routes panel instead of seeing the route displayed on the map.

### The Fix
Modified `handleRouteDetails` to:
1. Set `activeTab` to "map" when viewing route details
2. Close any open mobile panel (`setMobilePanelOpen(false)`)
3. Open the route detail panel (`setMobileRouteDetailOpen(true)`)

This ensures that when the user collapses the route details to see the map, they see the map with the route drawn on it, not the previous panel they came from.

## 9. Route Lines Persist Through Map Style Changes

### Overview
Fixed issue where route lines would disappear when changing map type (e.g., switching from Outdoors to Satellite) while viewing a route.

### The Problem
When the Mapbox style is changed, all sources and layers are removed by Mapbox. This caused:
- Selected route polylines to disappear
- Route creation lines to disappear
- Route pins to disappear

### The Fix
Implemented a `styleLoadCount` state that increments each time the map style finishes loading. This triggers all relevant `useEffect` hooks to re-add their sources/layers:

1. Added `map.once("style.load", handleStyleLoad)` listener
2. `styleLoadCount` dependency added to:
   - Route pins clustering useEffect
   - Selected route drawing useEffect
   - Creation route drawing useEffect
   - Route style update useEffect
3. Each useEffect now checks if sources/layers exist and re-creates them if needed
