## 1. Consistent Map/Options Navigation Pattern (Mobile)

### Overview
Implemented a consistent UX pattern across all mobile route tabs (Find Routes, Saved Routes, Create Route):
- When viewing a details panel, a **"Map" button** appears at the bottom to return to the map
- When viewing the map, an **"Options" button** appears to return to the relevant details panel

### Visual Pattern
```
DETAILS VIEW:                    MAP VIEW:
┌─────────────────────┐          ┌─────────────────────┐
│ [Header]            │          │ [Header]            │
├─────────────────────┤          │                     │
│                     │          │        MAP          │
│   Panel Content     │          │                     │
│   (filters, list)   │          │                     │
│                     │          │     [ Options ]     │
├─────────────────────┤          ├─────────────────────┤
│     [ 🗺️ Map ]      │          │   [Bottom Nav]      │
├─────────────────────┤          └─────────────────────┘
│   [Bottom Nav]      │
└─────────────────────┘
```

### Behavior
- **Map button**: Appears at bottom of detail panels (with gradient fade), closes panel and shows map
- **Options button**: Floats above bottom nav on map view, opens relevant panel
- State resets when switching tabs (always opens in panel view)

### Technical Implementation

**New State:**
```tsx
// Mobile panel state for Find/Saved (whether to show panel or map)
const [mobilePanelOpen, setMobilePanelOpen] = useState(true);
```

**Panel Props Added:**
- `mobilePanelOpen?: boolean` - Controls visibility on mobile
- `onMobilePanelToggle?: (open: boolean) => void` - Callback to toggle panel

### Files Modified

- Separated desktop (sidebar) and mobile (full-screen) layouts

**`components/routes/find-routes-panel.tsx`**
