# Changelog - 2026-03-06

## Session 1: Waypoint Edit Fixes & Photo Integration

**Summary:** Fixed multiple bugs in the waypoint editing system — save URL, tag/icon persistence, photo FK, photo display in timeline, and tag label consistency.

---

### Changes Made

#### 1. Fix Waypoint Save URL (404)

**Modified Files:**
- `components/routes/edit-waypoint-view.tsx`

**What Changed:**
- Fixed PATCH URL from `/api/waypoints/${id}` (non-existent) to `/api/routes/${routeId}/waypoints/${id}` (actual endpoint)
- This was why saving waypoint edits silently failed with 404

#### 2. Fix PATCH Endpoint null Handling

**Modified Files:**
- `app/api/routes/[id]/waypoints/[waypointId]/route.ts`

**What Changed:**
- Fixed PATCH endpoint to accept `null` for `icon_type` and `tag` fields
- Used `!== undefined` instead of truthy check so clearing a field works

#### 3. Fix Waypoint Photo Upload FK Error

**Modified Files:**
- `app/api/routes/[id]/waypoints/[waypointId]/photos/route.ts`
- `supabase/migrations/075_fix_waypoint_photos_fk.sql` (NEW)

**What Changed:**
- `waypoint_photos.user_id` FK referenced `auth.users(id)` instead of `public.users(id)`
- PostgREST join failed with 500 errors because it couldn't traverse the FK
- Removed explicit FK hint from join queries
- Migration 075 drops and re-creates FK pointing to `public.users(id)`

**Risk:** Migration 075 must be run in production Supabase SQL Editor.

#### 4. Fix Tag/Icon Not Persisting

**Modified Files:**
- `components/routes/edit-waypoint-view.tsx`

**What Changed:**
- Fixed state initialization from `useState("")` + useEffect to `useState(waypoint.prop || "")`
- Sending `tag: tag || null` and `icon_type: iconType || null` instead of empty strings
- Empty string `""` violated CHECK constraint `tag IN ('poi', 'instruction', 'caution', 'note')`

#### 5. Fix Tag Label Mismatch + Missing "Note" Tag

**Modified Files:**
- `components/routes/waypoint-card.tsx`

**What Changed:**
- Added missing `note` entry to `TAG_CONFIG` (was only instruction/poi/caution)
- Waypoints with `tag: "note"` now display "Note" badge on the card
- Changed `poi` label from "POI" to "Point of Interest" for consistency with edit form
- Styled "Note" tag with slate colors

#### 6. Fix Waypoint Photos Not Showing on Cards

**Modified Files:**
- `app/api/routes/[id]/waypoints/route.ts`

**What Changed:**
- API returned `community_photos` property but `waypoint-card.tsx` reads from `photos`
- Added `photos` as an alias of `community_photos` in the API response
- Waypoint cards in the timeline now display photo thumbnails

#### 7. Owner Photo Upload Without Completion

**Modified Files:**
- `app/api/routes/[id]/waypoints/[waypointId]/photos/route.ts`

**What Changed:**
- Route owner can upload photos to their own waypoints without completing the route
- Previously ALL uploads required route completion, blocking owner from adding photos during editing

#### 8. Hide Edit on Start/Finish Waypoints

**Modified Files:**
- `components/routes/waypoint-timeline.tsx`

**What Changed:**
- Added `!isStart && !isFinish` conditions to `onEdit` and `onSuggestEdit` callbacks
- Start and Finish are virtual waypoints derived from geometry, not DB records

#### 9. View All Waypoint Map Toggle

**Modified Files:**
- `app/routes/page.tsx`

**What Changed:**
- `handleEnterViewMode("waypoints")` now sets `showWaymarkers: true` in layer settings
- Added exit effect: waypoints view mode auto-exits when `showWaymarkers` toggled off
- Matches existing hazards pattern
