# Changelog - 2026-01-26

## Summary
Fixes for route editing, waypoint map improvements, and route completion ratings.

---

### 1. Fixed Edit Route Page TooltipProvider Error

**Why:** Clicking "Edit Route" caused a crash with error: `'Tooltip' must be used within 'TooltipProvider'`

**Files modified:**
- `app/routes/edit/[id]/page.tsx`

**Changes:**
- Added `TooltipProvider` import and wrapped ALL return statements (including loading state)
- The `RouteCreatorToolbar` uses Tooltip components which require this provider context

---

### 2. Fixed PathLayerToggles Props Mismatch

**Why:** Edit Route page crashed with error: `Cannot read properties of undefined (reading 'bridleways')` because the `PathLayerToggles` component was called with incorrect props.

**Files modified:**
- `app/routes/edit/[id]/page.tsx`

**Changes:**
- Updated `PathLayerToggles` call to use correct prop structure (`layers` object and `onToggle` callback)
- The component expects `{ layers: { bridleways, boats, footpaths, permissive }, onToggle }` not individual boolean props

---

### 3. Fixed Edit Route Page Layout and Functionality

**Why:** Edit Route page had sidebar on RIGHT (should be LEFT), missing toolbar, no snap/plotting modes, and didn't center on the route. Should work exactly like the Create Route page.

**Files modified:**
- `app/routes/edit/[id]/page.tsx` (complete rewrite)

**Changes:**
- Moved sidebar from RIGHT to LEFT (matching Create Route)
- Added full toolbar with plot/snap/undo/clear/style options
- Added proper waypoint management callbacks (add, insert, update, remove)
- Added `isCreating`, `isPlotting`, `snapEnabled` states to enable route editing on the map
- Map now centers on the route when loaded
- Layout now matches Create Route exactly

---

### 4. Added Nested Reply-to-Reply Feature (Instagram-style)

**Why:** Users could only reply to top-level comments. Now they can reply to replies with @mentions, creating Instagram-style conversation threads.

**Files modified:**
- `components/routes/route-detail-drawer.tsx`

**Changes:**
- Added "Reply" button to each reply in the discussion
- When replying to a reply, the reply text is automatically prefixed with `@username`
- All nested replies appear under the parent comment (flat structure with @mentions, like Instagram)
- Collapsible "View X more replies" / "Hide replies" toggles still work
- Reply input appears inline below the reply being responded to

---

### 5. Fixed Route Save Error (Difficulty Check Constraint)

**Why:** Saving a route failed with error `new row for relation "routes" violates check constraint "routes_difficulty_check"` because the PATCH endpoint wasn't mapping difficulty values.

**Files modified:**
- `app/api/routes/[id]/route.ts`

**Changes:**
- Added difficulty value mapping to PATCH endpoint (same as POST endpoint)
- Maps `"moderate"` → `"medium"`, `"difficult"/"severe"` → `"hard"`, `"unrated"` → `null`
- Database constraint only allows `easy`, `medium`, `hard`, or `null`

---

### 6. Changed "Create Route" to "Edit Route" When Editing

**Why:** The sidebar showed "Create Route" header even when editing an existing route.

**Files modified:**
- `components/routes/route-creator.tsx`

**Changes:**
- Added `isEditing` prop to `RouteCreatorProps` interface
- Header now shows "Edit Route" when `isEditing={true}`

---

### 7. Improved Edit Route Map Zoom

**Why:** When editing a route, the map didn't zoom in properly to show the route. Needed to fit the route bounds with appropriate padding.

**Files modified:**
- `app/routes/edit/[id]/page.tsx`

**Changes:**
- Added 20% padding to route bounds for better visibility
- Smaller routes get more zoomed in, larger routes more zoomed out (automatic)
- Default padding of 0.005 for very small routes to ensure visibility

---

### 2. Fixed WaypointMapPicker Not Showing Route Line

**Why:** When adding waypoints, the green route line wasn't visible on the map despite the geometry being present.

**Files modified:**
- `components/routes/waypoint-map-picker.tsx`

**Changes:**
- Added `routeGeometry` as direct dependency to the draw effect so it re-runs when geometry becomes available
- Added `zIndex: 1000` to ensure route polyline renders on top of other map elements
- Added console logging for debugging route drawing
- Fixed effect to properly wait for both map and geometry to be ready

### 3. Added Start/Finish Markers on Waypoint Map

**Why:** Users needed visual indicators for route start and end points when placing waypoints.

**Files modified:**
- `components/routes/waypoint-map-picker.tsx`

**Changes:**

---

### 4. Added Average Rating from Route Completions

**Why:** Users can rate routes when marking them complete, but this wasn't updating the route's average rating.

**Files created:**
- `supabase/migrations/061_route_completion_ratings.sql`

**Changes:**
- Created trigger `update_route_completion_rating_trigger` on `route_completions` table
- Automatically updates `routes.avg_rating` and `routes.review_count` when completions are added/updated/deleted
- Updates existing routes with completion ratings
- Added storage bucket policies for route photo uploads

---

### 5. Fixed Cover Images Not Showing on Explore Routes

**Files modified:**

**Changes:**

- Creates a cover photo map and merges with route data
