# Changelog - 2026-01-26

## Summary
Fixes for route editing, waypoint map improvements, and route completion ratings.

---

### 1. Fixed Edit Route Page TooltipProvider Error

**Why:** Clicking "Edit Route" caused a crash with error: `'Tooltip' must be used within 'TooltipProvider'`

**Files modified:**
- `app/routes/edit/[id]/page.tsx`

**Changes:**
- Added `TooltipProvider` import and wrapped ALL return statements (including loading state)
- The `RouteCreatorToolbar` uses Tooltip components which require this provider context

---

### 2. Fixed WaypointMapPicker Not Showing Route Line

**Why:** When adding waypoints, the green route line wasn't visible on the map despite the geometry being present.

**Files modified:**
- `components/routes/waypoint-map-picker.tsx`

**Changes:**
- Added `routeGeometry` as direct dependency to the draw effect so it re-runs when geometry becomes available
- Added `zIndex: 1000` to ensure route polyline renders on top of other map elements
- Added console logging for debugging route drawing
- Fixed effect to properly wait for both map and geometry to be ready

---

### 3. Added Start/Finish Markers on Waypoint Map

**Why:** Users needed visual indicators for route start and end points when placing waypoints.

**Files modified:**
- `components/routes/waypoint-map-picker.tsx`

**Changes:**
- Added green "S" marker at route start point
- Added red "F" marker at route finish point
- Markers have high zIndex (1001) to stay visible above route line

---

### 4. Added Average Rating from Route Completions

**Why:** Users can rate routes when marking them complete, but this wasn't updating the route's average rating.

**Files created:**
- `supabase/migrations/061_route_completion_ratings.sql`

**Changes:**
- Created trigger `update_route_completion_rating_trigger` on `route_completions` table
- Automatically updates `routes.avg_rating` and `routes.review_count` when completions are added/updated/deleted
- Updates existing routes with completion ratings
- Added storage bucket policies for route photo uploads

---

### 5. Fixed Cover Images Not Showing on Explore Routes

**Why:** Public routes weren't showing cover images on route cards.

**Files modified:**
- `app/api/routes/search/route.ts`

**Changes:**
- Added cover photo fetching for public routes (was only done for "My Routes")
- Creates a cover photo map and merges with route data

---
