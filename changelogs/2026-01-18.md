### 4. Routes Visibility RLS Fix

**Why:** Route detail drawer was closing immediately because routes couldn't be fetched - RLS policy used `visibility = 'public'` but routes were created with `is_public = true` (two different columns).

**File created:** `supabase/migrations/054_fix_routes_visibility_rls.sql`

**Changes:**
- Dropped conflicting RLS policies
- Created new unified policy that checks BOTH `visibility` and `is_public` columns
- Added owner access (owner can always see their routes)
- Added admin access (admins can see all routes)
- Syncs existing routes: sets `visibility` based on `is_public` value

**SQL to run:**
```sql
-- Run this in Supabase SQL Editor
DROP POLICY IF EXISTS "routes_visibility_select" ON routes;
DROP POLICY IF EXISTS "Anyone can view routes" ON routes;

CREATE POLICY "routes_visibility_select" ON routes
  FOR SELECT USING (
    visibility = 'public' 
    OR (visibility IS NULL AND is_public = true)
    OR is_public = true
    OR owner_user_id = auth.uid()
    OR EXISTS (SELECT 1 FROM users WHERE id = auth.uid() AND role = 'admin')
  );

UPDATE routes 
SET visibility = CASE 
  WHEN is_public = true THEN 'public'
  WHEN is_public = false THEN 'private'
  ELSE 'private'
END
WHERE visibility IS NULL OR visibility NOT IN ('public', 'private', 'link');
```

### 8. Fix Routes Search API - Ambiguous Relationship Error

**Error:** Routes were being created successfully (`POST /api/routes 201`) but not appearing in "My Routes" because the search API was failing with a 500 error.

**File modified:** `app/api/routes/search/route.ts`

**Changes:**
- Updated the `route_photos` join to use explicit FK hint: `route_photos!route_photos_route_id_fkey`
- This tells PostgREST which relationship to use when there are multiple FKs between tables

```typescript

route_photos (url, caption, order_index)

// After (explicit)
```
