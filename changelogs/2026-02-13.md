# Changelog — 2026-02-13

## Photo Upload, Reviews Display & Completions Count

### Session Summary

Implemented real photo upload in the review flow (Step 2), added a reviews display section in the route detail modal, and ensured completions count updates correctly when riders complete a route.

### Files Created

- `app/api/routes/[id]/completions/route.ts` — New GET endpoint returning completions with user info and parsed tags/notes
- `supabase/migrations/066_completions_count_trigger.sql` — Trigger to auto-increment/decrement `completions_count` on routes table + backfill

### Files Modified

- `components/routes/route-detail-drawer.tsx`
  - **Step 2 photo upload**: Replaced "Coming soon" placeholder with working upload — user selects a category tag (View, Feature, Seasonal, Horse, Hazard, Parking), picks a photo, sees preview, uploads to Supabase storage
  - **Photo tag badges**: Carousel shows category badge in top-left corner for tagged photos
  - **Reviews section**: New "Rider Reviews" section between Waypoints/Hazards and Engagement Stats — shows user avatar, name, time ago, review tags as pills, and short notes. First 3 shown with "Show all" button
  - **Fetch completions**: Added `fetchCompletions()` call on mount to load review data
  - Added state for photo upload (file, preview, category) and completions data
  - Added `validateRoutePhotoUpload` import and file input ref

- `app/api/routes/[id]/review/route.ts`
  - After inserting a completion, recounts all completions and updates `completions_count` on the routes table

- `app/api/routes/[id]/complete/route.ts`
  - Same completions count update after successful insert

- `app/globals.css` — (no new changes this session, scrollbar-hidden from earlier)

### What Changed and Why

1. **Photo upload** was showing "Coming soon" — riders can now contribute tagged community photos during the review flow. Tags help categorize photos (View, Hazard, Parking, etc.) and show as badges in the carousel.

2. **Reviews had no display** — the submission flow worked but reviews were never shown. Now completions with tags and notes display in a card-based section visible to all users.

3. **Completions count stayed at 0** — neither the review nor complete API handlers were incrementing the count. Added explicit count updates in both handlers, plus a SQL trigger for long-term reliability.

### Risk Notes

- SQL migration 066 must be run in Supabase dashboard for the trigger to take effect. Without it, the API handlers still update the count manually.
- Photo uploads go to the `route_user_photos` table for non-owner users (community photos). The existing photo API handles this correctly.
- The completions API endpoint parses JSON from the `notes` field — if notes contain non-JSON text, it falls back to treating it as plain text.
