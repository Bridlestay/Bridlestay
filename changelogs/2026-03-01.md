# Session 3 — Elevation Chart Line Quality Fixes (Evening)

## Summary
After user feedback that the elevation line looked "janky" (thick in some places, thin in others, broken appearance), performed deep UX examination identifying critical SVG rendering issues. Fixed non-uniform scaling artifacts, baseline misalignment, and stroke inconsistencies.

---

## Critical Issues Identified

### Issue 1: Janky Line Rendering (Non-Uniform Coordinate Warping)
**Problem:** Square viewBox (100×100) stretched to wide container (~6:1 aspect ratio) with `preserveAspectRatio="none"` caused non-uniform coordinate system warping. The 1.0px stroke appeared inconsistent due to SVG anti-aliasing artifacts.

**Root Cause:** Mismatched aspect ratios between viewBox and container dimensions.

### Issue 2: Misaligned Baseline Between Tabs
**Problem:** Line appeared to "start higher" in different tabs. Key point markers didn't align with the actual elevation line.

**Root Cause:** Key points calculated with 80% vertical scale, but line path used 65% vertical scale.

### Issue 3: Missing Non-Scaling Stroke
**Problem:** Stroke width scaled with warped coordinate system, creating visual inconsistencies.

**Root Cause:** Missing `vector-effect="non-scaling-stroke"` on line path.

---

**File:** `components/routes/elevation-profile.tsx`
**Lines:** 480

**Why:** Maintains same relative overlap detection distance for floating marker stagger logic with new viewBox scale.

---

## Technical Explanation

### SVG Coordinate System Warping

When an SVG has `preserveAspectRatio="none"`, the viewBox is stretched non-uniformly to fill the container. With our original square viewBox (100×100) in a wide container:

```
ViewBox:   100 units wide × 100 units tall (1:1)
Container: ~600px wide × ~100px tall      (6:1)
Result:    X-axis stretched 6×, Y-axis stretched 1×
```

This caused:
- **Stroke artifacts** — 1px stroke became ~6px horizontally but 1px vertically
- **Anti-aliasing issues** — SVG rendering engines struggled with the warped coordinate system
- **Visual inconsistency** — Line appeared thicker/thinner on different slopes

### Solution

New viewBox (400×100) better matches container aspect ratio:

```
ViewBox:   400 units wide × 100 units tall (4:1)
Container: ~600px wide × ~100px tall      (6:1)
Result:    X-axis stretched 1.5×, Y-axis stretched 1×
```

Much less warping = consistent stroke rendering.

Adding `vector-effect="non-scaling-stroke"` ensures the stroke width is calculated in screen pixels, not SVG coordinate units, eliminating any remaining inconsistencies.

# Session 4 — Floating Markers, Grid, and UI Polish (Late Evening)

## Summary
After deploying SVG viewBox fixes, user reported floating markers completely invisible (off-screen), area fill starting midway, grid barely visible, and harsh button/weather card styling. Fixed critical positioning bugs, enhanced grid visibility, and redesigned UI elements with soft Komoot-inspired aesthetic.

---

## Critical Issues Identified

### Issue 1: Floating Markers Positioned Off-Screen
**Problem:** Waypoints and hazards didn't appear in Waypoints/Hazards tabs. Drop-lines were missing.

**Root Cause:** Floating markers calculated `xPercent` on 0-400 scale for SVG coordinates, then applied it as CSS `%` positioning (`left: 200%` = way off screen). Lines 258, 284 used `* 400` for marker positioning, but CSS `left: ${m.xPercent}%` expected 0-100 scale.

**Solution:** Keep marker `xPercent` on 0-100 scale for CSS positioning, multiply by 4 when using in SVG.

### Issue 2: Area Fill "Starts Midway Up"
**Problem:** Elevation chart area fill had visible vertical line from bottom shooting up to first data point, creating "starts midway" appearance.

**Root Cause:** Path started at `M${firstX},100` (bottom), then `L${points[0]}` (first point midway up), creating unwanted vertical line.

**Solution:** Start path from first elevation point, not bottom.

### Issue 3: Grid Lines Too Subtle
**Problem:** Horizontal grid lines barely visible on warm off-white background.

**Root Cause:** Ultra-light gray (#D4D4D4), ultra-thin stroke (0.15px), low opacity (30%) made them nearly invisible.

**Solution:** Increase visibility with darker gray (#C4C4C4), thicker stroke (0.3px), higher opacity (40%).

### Issue 4: Harsh Button Styling
**Problem:** Hazard/warning buttons used bright red/amber borders and colors, clashing with soft Komoot aesthetic.

**Solution:** Redesign with soft slate gray, subtle borders, larger text, generous padding.

### Issue 5: "Disgusting" Weather Tips Card
**Problem:** Bright blue background/text clashed with muted earth tone palette.

**Solution:** Muted slate palette with Info icon, bullet points, better spacing.
